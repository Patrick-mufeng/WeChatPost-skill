# wechatpost-cover — 公众号封面设计

> 文章改完后做封面。参考 grindraft-cover 的 40 套模板 + 三维分析体系。

## 资源位置

- 40 套封面模板：`../wechatpost-skill/cover-templates/`
- 设计原则：`../wechatpost-skill/cover-templates/design-principles.md`
- 模板索引：`../wechatpost-skill/cover-templates/index.md`
- 截图脚本：`../wechatpost-skill/cover-templates/screenshot.js`（需 Node.js）

## 前置依赖

**HTML 预览**：无依赖，双击即开。
**PNG 渲染**（可选）：需要 Node.js + puppeteer + canvas。

---

## Workflow

```
用户: "做封面"
  ↓
Phase 0: 读取 final.md
  ↓
Phase 1: 三维分析（情绪/领域/调性）
  ↓
Phase 2: 推荐 3 个模板 → 用户选择
  ↓
Phase 3: 提取封面文案（标题/描述/标签）
  ↓
Phase 4: 生成预览 HTML
  ↓
Phase 5: 改稿循环（最多 3 轮）
```

## Phase 4 · 生成预览 HTML

输出到 `outputs/{标题}_{日期}/article/cover/`：

```
cover/
├── preview.html      ← 2.35:1 + 1:1 双版预览
├── cover-2x35.png    ← （需 Node.js，可选）
└── cover-1x1.png     ← （同上）
```

### 预览 HTML 规范

- Grid 布局：2.35fr 1fr，左右并排
- 所有 CSS 内联，无外部依赖
- 公众号标准尺寸：2.35:1 = 900×383，1:1 = 400×400
- 底部品牌栏：`<footer>` 标注 "WeChatPost · Design"，居中、字号 12px

### 设计规范（三个维度）

**色彩**：60-30-10 法则（主色60%+辅色30%+强调色10%），≤4色，对比度 ≥ 4.5:1

**字体**：
- 中文标题：无衬线黑体（PingFang SC / Microsoft YaHei）
- 英文标题：按模板风格（几何无衬线 / 衬线 / 等宽）
- 字号：标题 ≥ 48px，描述 ≥ 20px，标签 ≥ 14px

**布局**：
- 主标题居中或左对齐，占画面 40%
- 描述居标题下方，占 20%
- 标签放底部，占 15%
- 留白 ≥ 35%

## Phase 5 · 改稿循环

- 改文案 → 更新标题/描述/标签 → 刷新 HTML
- 改配色 → 微调
- "换一个风格" → 回到 Phase 2
- 最多 3 轮

---

## Phase 6 · 自检 — 逐项重读验证

| # | 目标 | 读什么 | 通过条件 |
|---|------|--------|----------|
| 1 | 预览 HTML | `cover/preview.html` | 文件存在，含 `2.35` 和 `1x1` 两个 `.cover-card` |
| 2 | HTML 内联 | 同上 | 所有 CSS 内联，无 `<link>` 或外部引用 |
| 3 | 封面文案 | 同上 | 含标题、描述、标签三个元素 |

```
📋 自检 Phase 6:
  □ cover/preview.html → {N} 字节，含双版预览 ✅
  □ CSS 全部内联 ✅
  □ 封面文案完整 ✅

✅ 封面自检全部通过
```

失败处理：缺失 → 重新生成 → re-read → 仍不通过 → ❌

---

## Key Rules

1. **只读 final.md 正文**——跳过 frontmatter
2. **HTML 预览零依赖**——双击即开
3. **一次一个风格**——不混搭
4. **配色克制**——≤4 色
