# 写作原则

继承自 grindraft 的公众号长文方法论。

## 五种文章原型

| 原型 | 特征 | 写法重心 |
|------|------|----------|
| **测评型** | 评测工具/产品/服务，有实测数据 | 体验过程→数据对比→结论推荐 |
| **工具型** | 分享实用工具/方法/Prompt | 痛点→工具展示→效果实测→使用指南 |
| **教程型** | 步骤拆解，可直接照做 | 材料清单→分步操作→成品展示→避坑提示 |
| **观点型** | 对热点/趋势发表独立判断 | 立场鲜明→对立理解→证据支撑→留白 |
| **故事型** | 人物/经历的叙事 | 场景切入→冲突展开→感悟收束 |

## 七条写作铁律

1. **开头从一个具体场景/事件切入**——不写"在当今 AI 快速发展的时代"，写"昨天下午我发现..."

2. **像聊天不像写报告**——句子时长时短，段落短（2-4句为主），偶尔一句话独立成段

3. **知识是"聊着聊着掏出来的"**——不写"下面我来介绍一下"，写"说到这个，有个事挺有意思的"

4. **有立场有态度**——敢下判断，但不居高临下——"我觉得""我个人认为"

5. **用具体的而不是空泛的**——不说"AI工具"，说"Claude Code"；不说"最近"，说"上周三"

6. **给读者一个"人"**——如果有数据点，用人物画像法把数据变成具体的人

7. **扣主线句高频出现**——每展开一个子话题后，用一句话拉回主线

## 结构模板

```
【开头】感性切入，从一个具体事件/场景开始，迅速建立情绪
  ↓
【背景铺垫】聊天式科普，让非专业读者也能看懂
  ↓
【核心内容】分板块展开，每板块有：
  - 一个明确观点
  - 至少一个具体场景/细节支撑
  - 一句扣主线的话
  ↓
【升华】从具体事件拉到更大的视角/思考
  ↓
【收尾】回环呼应或行动呼吁——不强求"完美收束"，允许留白
```

## 四层去AI味自检

### L1 硬规则扫描（零容忍）

| 禁用 | 替换方向 |
|------|----------|
| "首先...其次...最后" | 用自然转场替代 |
| "综上所述" / "总的来说" | 删掉，或用具体回扣句 |
| "值得注意的是" / "不难发现" | 删掉，直接说 |
| "在当今...的时代" / "随着...的发展" | 改成具体场景切入 |
| "让我们来看看..." / "接下来让我们..." | 删掉，直接说 |
| "不可否认" / "众所周知" | 删掉，直接陈述 |
| 结尾喊口号（"让我们一起..."） | 改成具体建议或留白 |

### L2 风格一致

- 开头是具体场景切入？
- 长短句交替？连续3句长度相近=节奏呆板
- 至少5个不同口语化过渡词（"说实话""你想想看""问题来了""说白了就是"...）
- 板块之间口语化转场，非纯markdown小标题堆叠
- 结尾避免口号，回扣开头或留白

### L3 内容质量

- 每个观点有具体细节/场景/案例支撑？
- 知识点是"聊天式掏出"还是"教科式介绍"？
- 有对立面理解？先站到读者处境再给视角？

### L4 活人感终审

核心问题：**"读完这篇文章，感觉是一个有见识的人在认真跟我聊，还是一个 AI 在给我输出信息？"**

- 温度感：情绪是体感记忆（"我当时就愣住了"）还是知识描述（"我感到非常震撼"）？
- 独特性：换一个人+换一个 AI 也能写出差不多的？
- 姿态："有见识的普通人在聊一件打动他的事"，而非"导师在教学生"

## 标题生成

> 核心理念：**克制但高点击**。不夸大、不标题党，但让读者产生"想看"的冲动。
> 每篇生成 5 个候选，用户从候选中选择。

### 六套标题公式

写标题时按以下 6 套公式各生成 1-2 个候选，覆盖不同点击诱因：

| # | 公式 | 骨架 | 公众号适配示例 |
|---|------|------|---------------|
| 1 | **数字清单型** | 数字 + 角度 + 价值点 | "Claude Code 的 3 个隐藏用法，第 2 个太省事了" |
| 2 | **对比冲突型** | A vs B + 结论暗示 | "用 Whisper 转录 vs 人工听写，差距不止 10 倍" |
| 3 | **结果导向型** | 人群 + 收益 + 方式 | "想做视频转文章？这份全流程指南请收好" |
| 4 | **场景共鸣型** | 当你在...时，其实... | "每次改稿改到凌晨，其实你缺的不是时间" |
| 5 | **反常识型** | 你以为...其实... | "为什么越懂 AI 的人，越不急着用 AI 写文章？" |
| 6 | **工具模板型** | 我把...整理出来了 | "我把视频转录→发公众号的全流程整理成了模板" |

### 五维评分

从公式池中选出 5 个最优候选，按以下维度打分：

| 维度 | 说明 | 权重 |
|------|------|------|
| 钩力 | 前5字是否制造信息缺口或认知冲突 | 30% |
| 清晰度 | 读者是否一眼看懂文章在讲什么 | 25% |
| 情绪值 | 是否引发好奇、共鸣或紧迫感 | 20% |
| 关键词 | 是否自然含搜索关键词 | 15% |
| 差异化 | 与同话题竞品标题的区别度 | 10% |

综合分 = 钩力×0.3 + 清晰度×0.25 + 情绪×0.2 + 关键词×0.15 + 差异×0.1

### 质量过滤（硬规则）

评分前先过以下四道滤网，不通过的标题直接淘汰或改写：

| # | 规则 | 说明 |
|---|------|------|
| 1 | **≤30 字** | 公众号标题超 30 字会被截断，严格控制 |
| 2 | **不编造数据** | 无可验证的真实数字时，用"明显/显著/可观"替代具体百分比；禁止凭空写"提升 30%" |
| 3 | **禁用过度情绪词** | 禁用"救命/绝了/哭死/必看/收藏！/不看后悔"等自媒体浮夸用语；保持克制、可信 |
| 4 | **对象 + 场景 + 收益** | 每个标题至少暗示三要素：谁（读者画像）、在什么情况下、能得到什么。缺要素 → 补上或淘汰 |
