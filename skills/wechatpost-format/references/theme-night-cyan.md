# 暗夜青 设计语言

> 本文件是「暗夜青」主题的**设计语言定义**，非组件模板。目标是让 AI 理解这套设计语言后，为每篇文章进行**原创排版设计**。下文组件描述为设计原则与灵感参考，AI 应根据文章内容自由演绎，而非逐字复制。

## 设计原则与气质

### 气质关键词

科技、终端美学、霓虹克制、未来感、数据感

### 配色哲学

暗夜青的配色逻辑是「深空为底，青霓为星，灰阶为尘」。全文底色 `#1a1a2e`——不是纯黑，而是深空蓝紫调，像深夜终端屏幕的背景色。正文使用浅灰 `#d0d0d0`，在深底上保持柔和可读（纯白 `#fff` 在深底上对比度过高会产生光晕效应，刻意避免）。主色 `#00d4aa` 青绿色只在需要「点亮」时出现——它像仪表盘上的指示灯，用量极少但每次出现都有信号意义。

这套色板的关键约束是**霓虹克制**：青色不是装饰涂料，是发光二极管。全文青色的覆盖面积不应超过总像素的 3%。它出现在章节编号、代码块边框、关键数字、按钮上——每次出现都在说「这里是数据/这里是信号/这里值得注意」。其余 97% 的内容由灰阶（`#d0` / `#808090` / `#5a5a70`）承载，在深色基底上形成三层灰度深度。

### 视觉节奏

- **段落间距**：段落间 margin 8-10px。暗色底上文字天然比白底需要更多的间距来维持可读性——如果极简蓝的 8px 是「正常呼吸」，暗夜青的 8px 就是「最低呼吸」。
- **焦点密度**：每屏 1-2 个青色信号。青色是稀缺资源——不要在同一屏内出现章节编号青色 + 数据数字青色 + 代码边框青色三者叠加（选 1-2 个即可）。
- **留白策略**：暗夜青的留白不是「白色」，是「深空色」——它不是空白，是深度。章节间 32px 的深空留白，制造「穿过一片黑暗到达下一个信息节点」的体验。组件之间 20-24px。
- **节奏控制**：暗色底天然让阅读节奏降低 20%——读者在暗色底上的阅读速度比白底慢。因此暗夜青的段落应该更短（4-5 行而非 5-6 行），让读者更频繁地「换气」。

### 强弱规则

| 层级 | 名称 | 视觉表现 | 频率限制 |
|------|------|---------|---------|
| 锚点层 | 青色信号 | 章节编号、代码边框 `1px solid #00d4aa`、关键数字、按钮底色、下划线 | 每屏 ≤2 处 |
| 标记层 | 加粗/半透明底 | 正文加粗（浅灰 bold）、卡片半透明底 `#252540`、行内代码青底 | 每段 ≤2 处加粗 |
| 容器层 | 深紫灰边框/深底 | 卡片 `#252540` + `1px solid #2d2d50`、分割线、图片容器 | 无限制 |

原则：暗夜青不使用纯黑——最深的基底是 `#1a1a2e`，最深的卡片是 `#16162a`（代码块底），它们之间形成微妙的深度差（像终端的多层窗口叠加）。青色永远是信号灯，不是氛围灯。

## 组件设计模式

> 以下为设计指导，非固定模板。AI 应根据具体文章特点，在这些约束边界内自由创造。

### 1. 封面/头部信息卡

**设计方向**：命令行档案式——像在终端里 `cat` 了一篇文章的元信息。

- 卡片底色：`#252540`（半透明深紫底），边框 `1px solid #2d2d50`，圆角 8px。
- 顶部仿终端提示符：`$ cat article-meta.json` 或 `$ ./info --summary`，使用等宽字体，`font-size:10px;color:#808090;letter-spacing:1px`。这是暗夜青独有的签名式元素。
- 标签：3-5 个青色边框标签——`border:1px solid #00d4aa;color:#00d4aa;font-size:9px;padding:2px 8px;border-radius:3px;letter-spacing:1px`（镂空青边框，不用实心药丸——实心在暗色底上太重）。
- 中部：文章标题 `font-size:20-22px;color:#e8e8f0;font-weight:bold`。
- 底部：字数 + 阅读时间 + 一句话概括。元数据用 `#808090` 10px，概括句用 `#d0d0d0` 12px。
- 变体方向：可用纯深底 `#1a1a2e` 无卡片仅左竖条（3px 青色）；可在标题前加一个青色小方块 `■` 做装饰；终端提示符可用不同的命令（适应文章主题）。
- 约束：不使用渐变；不使用阴影；终端提示符为可选（不是每篇都需要）。
- **封面设计自由度**：暗夜青的封面应保持「终端即视感」：
  - **命令行档案式**（默认）：`$ cat article-meta.json` + 标签 + 标题 + 元数据。
  - **系统信息式**：用 `--- SYSTEM INFO ---` 等终端横幅包装元数据，适合纯技术文章。
  - **仪表盘式**：标题上方加一行 `[STATUS: ONLINE]` 伪系统状态行，`font-family:monospace;color:#00d4aa;font-size:9px`。
  - AI 应根据文章气质选择——技术教程用命令行档案式，开发技术用系统信息式，数据报告用仪表盘式。

### 2. 章节标题

**设计方向**：青色编号 + 标题，左侧可选竖条。

- 结构：青色编号（如 `01` 或 `[01]`） + 标题文字。
- 编号：`font-size:14px;color:#00d4aa;font-weight:bold;letter-spacing:2px;font-family:monospace`。用等宽字体编号——像终端里的行号。
- 标题：`font-size:18px;color:#e8e8f0;font-weight:bold;margin-top:4px`。
- 变体方向：可在编号和标题之间加一条 3px 青色竖条 `background:#00d4aa`；编号可用 `> 01` 的 shell 风格前缀。
- 约束：章节标题上方 margin 32px；编号必须使用等宽字体（区别于正文的非等宽字体——这是暗夜青的「代码感」）。

### 3. 正文段落

**设计方向**：灰字深底，简洁直接。

- 基准：`font-size:14px;color:#d0d0d0;line-height:1.85;letter-spacing:0.3px;text-align:justify`。
- 首段处理：推荐首段加一条微型装饰——左侧 2px 青色竖条 `border-left:2px solid #00d4aa;padding-left:10px`，但文字颜色不变。暗示「从这里开始阅读」。也可不做特殊处理。
- 段落间距：基准 8px，长段落可放宽到 10px。
- 约束：每段不超过 5 行（暗色底阅读疲劳更快到来）；段落间不添加任何装饰符号。

### 4. 行内标记

**设计方向**：青灰二元标记，青色仅在「信号」位置。

- **加粗**：`<strong>` 直接包裹，颜色保持 `#e8e8f0`（比正文浅一阶，起强调作用）。
- **高亮/下划线**：使用 `border-bottom:2px solid #00d4aa;font-weight:600`（青色下划线，暗色底上非常醒目）。这是本主题关键词标记的唯一方式。
- **行内代码**：深底 `background:#252540;color:#00d4aa;padding:1px 6px;border-radius:4px;font-family:monospace;font-size:13px;border:1px solid #2d2d50`。行内代码加了一条极细边框，让它从深色背景中浮起来。
- **删除线**：`text-decoration:line-through;color:#5a5a70`。
- 约束：同一段落行内代码不超过 2 处（暗底上太多边框碎片会分散注意力）；下划线标记每屏不超过 2 处。

### 5. 引用/金句块

**设计方向**：终端输出式 + 半透明卡片式。

- **变体 A — 终端输出式（默认，暗夜青独有）**：青色 `>` 前缀 + 文字。像终端里的输出行。`color:#00d4aa;font-family:monospace` 的 `>` 符号，后跟引用文字 `font-size:15px;color:#e8e8f0;font-weight:bold`。无底色、无边框——就是一行终端输出。
- **变体 B — 半透明卡片（强调包裹）**：`background:#252540;padding:14px 16px;border-radius:8px;border:1px solid #2d2d50`。文字 `font-size:15px;color:#e8e8f0`。像终端里的高亮输出块。
- **变体 C — 左侧青色竖条（经典克制）**：`border-left:3px solid #00d4aa;padding:6px 0 6px 12px;background:transparent`。用于长段引用。
- 选用原则：短金句或代码相关引用用 A；核心结论用 B；严肃引用/长者言论用 C。
- 约束：变体 A 中的 `>` 符号必须等宽字体且青色；同一文章中变体 A 和 B 不同时出现在同一屏。
- **金句提取策略**：暗夜青的金句提取标准偏向「技术洞察」和「数据结论」：
  - 包含「关键」「核心」「本质」等判断词的句子
  - 以数据结尾的结论句（如「性能提升 3 倍」）
  - 包含代码术语或技术名词的短句（如「recursion is the key」）
  - 段落的结论句（通常是末句）
  提取后，技术金句用变体 A（终端 `>` 前缀），核心结论用变体 B（半透明卡），长引用用变体 C（竖条）。

### 6. 提示/旁注块

**设计方向**：半透明深底 + 青色左边框，像 IDE 里的 info 面板。

- 结构：`border-left:3px solid #00d4aa;background:#252540;padding:10px 14px;border-radius:0 8px 8px 0`（左侧青色竖条，右侧圆角）。
- 文字：`font-size:13px;color:#d0d0d0;line-height:1.75`。
- 顶部可选：`INFO` 或 `NOTE` 小标签，使用 `font-size:9px;color:#00d4aa;letter-spacing:2px;font-family:monospace`。
- 变体方向：可用四周 `1px solid #2d2d50` 框代替左侧竖条；可在左侧放一个青色小方块 `■` 代替竖条。
- 约束：提示块内不超过 3 行；同一屏不超过 1 个。

### 7. 数据/步骤展示

**设计方向**：仪表盘式数据卡 + 终端编号步骤。

- **数据展示**：三列 flex 卡片，深紫底 `#252540`，边框 `1px solid #2d2d50`。大数字用 `#00d4aa` bold 28px，标签用 `#808090` 10px。数字是仪表盘上的关键指标——每张数据卡像一个监控面板的 widget。
- **步骤（横向）**：圆形青色编号——关键选择：可使用镂空圆（`border:2px solid #00d4aa;color:#00d4aa`）而非实心圆。镂空在暗底上更轻盈、更有「未点亮但已就绪」的科技感。
- **步骤（纵向）**：编号左对齐，步骤间用 `border-left:1px solid #2d2d50` 连接（时间轴效果，暗底上竖线比横线更自然）。
- **表格数据**：flex 模拟行，偶数行 `#252540` 底色，表头青字 `#00d4aa` + 深底 `#1a1a2e`。
- 约束：卡片圆角 8px；编号镂空/实心全文统一（二选一）；数据数字保持青色。
- **数据可视化决策树**：
  - 单一核心数据 → 居中大字展示（`font-size:36px;color:#00d4aa;font-weight:bold;font-family:monospace`）
  - 2-3 个对比数据 → 三列深紫底卡片 + 青色数字
  - 4+ 数据行 → 表格（flex 模拟，偶数行 `#252540`）
  - 流程/步骤 → 纵向步骤流 + `#2d2d50` 竖线连接
  - 时间线 → 纵向步骤流，圆形标识用镂空圆（`border:2px solid #00d4aa`）

### 8. 图片与媒体

**设计方向**：深色容器 + 细边框，像暗色模式下的图片查看器。

- 标准图片：深底容器 `#1a1a2e` + 边框 `1px solid #2d2d50`。不使用阴影——暗底上阴影看不见且无意义。
- 圆角策略：图片容器 `border-radius:8px`，与卡片统一。
- 图片说明：`font-size:11px;color:#808090;text-align:center;margin-top:6px`。
- GIF 角标：深紫底 `#252540` + 青色文字 `#00d4aa` 胶囊标签，`border:1px solid #2d2d50`。
- 约束：图片上下间距 12-16px；不使用虚线边框。

### 9. 代码块

**设计方向**：终端原生风格——这是暗夜青的**灵魂组件**。

- 深底 `#16162a`（比文章底更暗一阶，造出「终端窗口沉在屏幕里」的深度），顶栏 `#0f0f1f` + 三色终端圆点。
- 代码文字 `#d0d0d0`，等宽字体栈。语法高亮使用柔和的霓虹色系：关键字 `#c678dd`（紫）、字符串 `#98c379`（绿）、函数 `#e5c07b`（黄）、注释 `#5c6370`（灰）。
- 边框：`border:1px solid #2d2d50`（微妙边框让代码块在深底上不被吞没）。
- 顶栏可选：语言标签（`font-size:9px;color:#808090;font-family:monospace`）或终端路径（`~/project $`）。
- 行内代码：深底 `#252540` + 青色 `#00d4aa` + 细边框 `1px solid #2d2d50`。
- 约束：**代码块必须使用深色终端风格**（不得使用浅色代码块——在暗色文章中使用浅色代码块是视觉灾难）；代码边框为可选，但如果全文首次出现代码块建议加边框。

### 10. 分割线

**设计方向**：符号装饰型——青色 `◆ ◆ ◆` 或终端风格 `----`。

- 样式 A（终端风格）：`- - - - -`，`color:#2d2d50;letter-spacing:4px;font-family:monospace`。等宽字体的横线连接成一条虚线。
- 样式 B（霓虹符号）：`◆ ◆ ◆`，`color:#00d4aa;letter-spacing:6px;font-size:11px`。青色菱形符号——暗夜青最华丽的视觉元素，但保持小尺寸不抢夺注意力。
- 样式 C（极简灰线）：1px `#2d2d50` 居中短线段，宽 40%-60%。
- 选用原则：技术文章用 A；数据报告用 B；一般分隔用 C。
- 约束：样式 B 的菱形符号每篇文章最多出现 2 次（物以稀为贵）；分割线不使用纯白色。

### 11. 尾部互动卡

**设计方向**：终端登出式——像 `exit` 了一个会话。

- 卡片底色：`#252540`，边框 `1px solid #2d2d50`，圆角 8px。
- 顶部：仿终端提示符 `$ exit` 或 `$ logout`，`font-family:monospace;font-size:10px;color:#808090`。
- 引导文案：`font-size:13px;color:#e8e8f0;text-align:center`。示例：「session ended. 感谢阅读」「如果对你有帮助」——暗夜青可以用技术语言做情感表达。
- 互动按钮：推荐风格 A 经典按钮——青色实心底 `#00d4aa` + 深色文字 `#1a1a2e`（青色底上放深色文字而非白色，降低亮度冲击）。或风格 E 暗色标签卡——青色边框+青色文字镂空按钮。
- 可选：结束语前加一行 `— END OF TRANSMISSION —` 或 `— EOF —`，`font-family:monospace;font-size:9px;color:#5a5a70;letter-spacing:2px`。
- 结束语：`font-size:10px;color:#808090;text-align:center`。像终端会话结束后的提示行。
- 约束：尾部卡片高度不超过 200px；互动按钮不超过 3 个；不使用实心白色按钮。

## 文章类型到设计策略

| 文章类型 | 设计策略 | 节奏特点 | 重点组件 |
|---------|---------|---------|---------|
| 技术教程 | **暗夜青黄金场景**。终端代码块全开；步骤用纵向终端编号；章节标题清晰；代码即内容 | 技术文档节奏，代码块穿插频繁 | 代码块（终端风格）、步骤流、章节标题、行内代码 |
| 工具测评 | 数据卡仪表盘式排列；图片占比高（截图为深色底天然融入）；标签青色镂空 | 信息密度中高，数据卡如监控面板 | 数据卡片、图片容器、标签（镂空）、引用块 B |
| 知识整理 | 章节标题主导结构；提示块如 IDE info 面板；大量使用步骤流 | 结构化节奏，章节间大分隔 | 章节标题、提示块、步骤流（纵向）、引用块 C |
| 观点分析 | 减少终端元素（不用 `$` 前缀），增加半透明卡片包裹观点；青色仅用于最核心的 1-2 句 | 慢节奏，观点句用卡片突出 | 引用块 B、提示块、章节标题、分割线 C |
| 深度文章 | 暗夜青的「纯阅读模式」——减少青色信号到全文 3-4 处；正文灰阶主导；节奏极慢 | 极慢节奏，深空留白制造沉浸感 | 章节标题、引用块 C、正文段落、分割线 C |

## 快速气质校验清单

排版完成后，AI 应自查以下问题：

- [ ] 全文底色是否为深空色 `#1a1a2e`（非纯黑 `#000`）？
- [ ] 正文是否为浅灰 `#d0d0d0`（非纯白 `#fff`）？纯白在深底上对比度过高？
- [ ] 青色 `#00d4aa` 是否仅出现在信号位置（编号/边框/数据/按钮/下划线）？总覆盖面积 ≤3% 像素？
- [ ] 每屏青色元素不超过 2 处？
- [ ] 卡片底色是否为半透明深紫 `#252540`？边框是否为 `#2d2d50`？
- [ ] 代码块底色是否为 `#16162a`（比文章底更深）？是否全部使用深色终端风格？
- [ ] 章节编号是否使用等宽字体（区别于正文的非等宽）？
- [ ] 图片容器是否使用了 `#2d2d50` 边框（而非无边框被深底吞没）？
- [ ] 尾部互动卡是否使用青色实心底 + 深色文字（而非白字）？
- [ ] 所有中文文字是否 `<span leaf="">` 包裹？
- [ ] 段落是否控制在 4-5 行以内（暗底上更短的段落节奏）？

## 常见设计陷阱与避免模式

以下是 AI 在使用暗夜青主题进行排版时最常犯的错误，请逐条对照检查：

**陷阱 1：纯黑底色**——使用 `#000` 或 `#111` 作为文章底色，而非深空色 `#1a1a2e`。纯黑缺乏深度感，让暗夜青变成「黑底白字」的粗糙方案。深空蓝紫调 `#1a1a2e` 是精心调配的暗色基底——它在黑色中注入了微量的蓝紫，制造深邃的空间感。

**陷阱 2：白色正文**——使用 `#fff` 或 `#f0f0f0` 作为正文色。暗底上的纯白文字会产生光晕效应（halation），长时间阅读会导致视疲劳。正文必须使用浅灰 `#d0d0d0`——足够亮以保证可读性，又足够柔以避免刺眼。

**陷阱 3：青色霓虹滥用**——在标签、图标、装饰线、分割线、渐变、多处按钮等位置大量使用 `#00d4aa`。青色是暗夜青的激光指示器——全文青色覆盖面积应不超过总像素的 3%。如果一个读者扫过全文时感觉「到处都是绿色」，就是失败了。

**陷阱 4：浅色代码块**——在暗底文章中使用浅色代码块（`#f6f8fa` 等白底）。这是排版灾难——暗底文章中突然出现大白块会刺眼且破坏沉浸感。代码块必须使用深色终端风格 `#16162a`。

**陷阱 5：无边框卡片被吞没**——卡片使用 `#252540` 底但忘了加 `border:1px solid #2d2d50`。暗底上的深色卡片如果不加边框，边缘会融入文章底色而「消失」。那条 1px 的 `#2d2d50` 边框是卡片存在的证明。

**陷阱 6：非等宽编号**——章节编号使用了非等宽字体（如系统默认字体）。暗夜青的「代码感」依赖等宽字体作为视觉标识——编号、终端提示符、代码块都必须使用等宽字体栈。

**陷阱 7：尾部白字按钮**——青色实心底 `#00d4aa` 的按钮上使用了白色文字 `#fff`。青+白在深底上亮度冲击过大。应使用深色文字 `#1a1a2e` 放在青色底上，保持克制。

**陷阱 8：整段都是加粗**——暗底上为了提升可读性而将整段都设为 `<strong>`。暗夜青的正文加粗应该用 `#e8e8f0` 且仅用于关键词——加粗太多反而让读者失去方向。

## 排版决策流程速查

AI 拿到一篇 Markdown 文章后，按以下决策链进行排版设计：

```
1. 扫描文章，判断文章类型 → 查「文章类型到设计策略」表确定节奏和重点组件
2. 设置全文底色 → #1a1a2e（深空色），这是第一步，不可省略
3. 扫描文章，提取金句 → 按内容性质选变体（技术金句→A 终端 > / 核心结论→B 半透明卡 / 长引用→C 竖条）
4. 扫描文章，识别数据点 → 按「数据可视化决策树」选展示形式
5. 生成封面信息卡 → 根据文章气质选方向（技术→命令行档案式 / 开发→系统信息式 / 数据→仪表盘式）
6. 遍历章节生成章节标题 → 青色等宽编号+标题，可选左侧竖条
7. 遍历段落生成正文 → 14px 浅灰 #d0d0d0，行高 1.85，每段 4-5 行
8. 遇到代码块 → 始终深色终端 #16162a，可选边框 1px #2d2d50
9. 遇到图片 → 深底容器+细边框 #2d2d50，无阴影
10. 生成分割线 → 终端虚横线 / 青色 ◆ ◆ ◆ / 灰线
11. 生成尾部互动卡 → 青底+深色文字按钮，终端登出式卡片
12. 全文自检 → 对照「快速气质校验清单」
```
