# 暖纸墨 设计语言

> 本文件是「暖纸墨」主题的**设计语言定义**，非组件模板。目标是让 AI 理解这套设计语言后，为每篇文章进行**原创排版设计**。下文组件描述为设计原则与灵感参考，AI 应根据文章内容自由演绎，而非逐字复制。

## 设计原则与气质

### 气质关键词

温度、杂志感、细线分隔、人文、翻阅感

### 配色哲学

暖纸墨的配色逻辑是「纸色为底，墨色为字，陶土为饰」。全文底色 `#fdfaf4`——不是纯白，而是微黄的纸张色，让读者在电子屏幕上有翻阅实体杂志的错觉。正文和标题统一使用深棕 `#4a3728`，比纯黑柔和，却比灰色有分量。主色 `#b8532a` 暖陶土色出现在需要「划线标记」的地方——大引号、章节编号、竖条、按钮。

这套色板的本质是**减法**：仅用 3 种功能性色彩（纸底色 + 墨色字 + 陶土标记），其余全部靠线条和留白分层。如果极简蓝是蓝色做路标，暖纸墨就是陶土色做「编辑的红笔」——批注感、圈点感、手工感。

### 视觉节奏

- **段落间距**：段落间 margin 8-10px，但段落与段落之间靠「段落内在的内容节奏」驱动，而非硬性间距。长段落后可跟较大的间距（12px）作为呼吸点。
- **细横线（hairline）节奏**：这是本主题最核心的视觉语言。`1px solid #e8d5c4` 细横线用作章节间的大分隔（占满宽），也用作小节间的短分隔（居中 60% 宽度）。线的粗细始终 1px，颜色始终是浅边框色——它们像杂志排版中的栏线，不抢眼但不可或缺。
- **焦点密度**：每屏 1-2 个陶土色锚点（大引号、编号、竖条）。陶土色出现的频率要低于极简蓝的蓝色——暖纸墨更接近「阅读」而非「扫描」，读者不需要太多路标。
- **留白策略**：暖纸墨的留白不追求机械均匀——它像杂志排版那样，有时一段文字占据大半屏，有时一句话独占一行。留白的「不规则感」恰恰是暖纸墨的温度来源。

### 强弱规则

| 层级 | 名称 | 视觉表现 | 频率限制 |
|------|------|---------|---------|
| 锚点层 | 陶土色标记 | 大引号、章节编号、左侧 3px 竖条、按钮底色、关键词下划线 | 每屏 ≤2 处（比极简蓝更少） |
| 标记层 | 加粗/暖底 | 正文加粗（深棕 bold）、卡片暖纸底 `#f7f0e6`、行内墨色底代码 | 每段 ≤2 处加粗 |
| 容器层 | 细线/纸色 | 细横线、卡片暖底色、图片细边框、分割线装饰 | 细横线每章间 1 条 |

原则：暖纸墨的强弱层次比极简蓝更扁平——它的「设计感」来自于细线和纸色，而非色彩对比。如果不加细横线，全文几乎只有一种颜色（深棕）。正是这条 1px 的线，让平淡的纸面有了结构。

## 组件设计模式

> 以下为设计指导，非固定模板。AI 应根据具体文章特点，在这些约束边界内自由创造。

### 1. 封面/头部信息卡

**设计方向**：杂志目录式——像翻开一本设计杂志的第一页。

- 卡片底色 `#f7f0e6`（暖纸浅底），上下用 1px `#e8d5c4` 细横线框住（顶部一条线 + 底部一条线）。
- 顶部线之上：可选装饰性文字（如 `VOL.01` 或 `ESSAY`），用小号 9px 浅棕 `#b8a898`，letter-spacing:3px。
- 卡片内部：上方 3-5 个话题标签——使用陶土色文字 `#b8532a` + 浅棕边框 `border:1px solid #e8d5c4` 的镂空标签（不用实心药丸，保持轻盈）。
- 中部：文章标题 `font-size:20-22px;color:#4a3728;font-weight:bold`。
- 底部：字数 + 阅读时间（`font-size:10px;color:#b8a898`）+ 一句话概括（`font-size:12px;color:#8b7355;line-height:1.7`）。
- 变体方向：可不用暖底仅用上下细线框；可在标题左侧用一条 3px 陶土色竖条做标记；可加一行装饰性小圆点 `· · ·` 在概括句上方。
- 约束：信息卡左右留白不少于 8px；不使用渐变；不使用阴影；标签不超过 5 个。
- **封面设计自由度**：暖纸墨的封面可以更有杂志感：
  - **目录式**（默认）：上下细横线框 + 标签 + 标题 + 元数据，如杂志目录页。
  - **引语式**：标题下方用大号斜体引语（`font-size:15px;color:#8b7355;font-style:italic`）代替概括句——像杂志的副标题。
  - **纯标题式**：去除所有装饰，仅标题 + 下方一条细横线。适合深度人文文章。
  AI 应根据文章气质选择——观点分析用目录式，深度文章用纯标题式，人文思考用引语式。

### 2. 章节标题

**设计方向**：细横线 + 编号 + 标题，像杂志的章节扉页。

- 结构：上方一条细横线 `border-top:1px solid #e8d5c4`（margin-bottom:16px），下方是编号 + 标题。
- 编号：使用陶土色 `#b8532a`，`font-size:12px;font-weight:bold;letter-spacing:2px`。格式用 `01.` 或 `/ 壹 /`（中文编号可选，增强杂志感）。
- 标题：`font-size:18px;color:#4a3728;font-weight:bold;margin-top:4px`。
- 变体方向：可将细横线放在章节标题之下（作为底线）；可添加左侧 3px 陶土色竖条替代顶部细线（保持与极简蓝的差异化——有竖条时不用顶部横线）；编号可用罗马数字。
- 约束：细横线颜色固定 `#e8d5c4`，宽度 1px；章节标题上方 margin 32px；每个章节标题必须有编号。

### 3. 正文段落

**设计方向**：墨色文字，纸色底，像印刷品。

- 基准：`font-size:14px;color:#4a3728;line-height:1.85;letter-spacing:0.3px;text-align:justify`。
- 首段处理：推荐首段做一句倾斜导语（`font-style:italic;color:#8b7355;font-size:15px`），分隔整篇文章的「导入」和「正文」。
- 段落间距：基准 8px。暖纸墨的段落间距可以有弹性——如果前一段以问句结尾，下一段给出答案，间距可以压缩到 6px 制造紧凑感；如果话题转换，放宽到 12px。
- 约束：每段不超过 6 行（约 240 字）；段落间不使用任何装饰符号（细横线是章节级的，不出现在段落间）。

### 4. 行内标记

**设计方向**：墨纸二元标记，陶土色仅用于最关键的指向。

- **加粗**：`<strong>` 直接包裹，颜色保持正文 `#4a3728`。加粗是编辑的钢笔划重点，不加颜色。
- **高亮/下划线**：使用 `border-bottom:2px solid #e8c4a2;font-weight:600`（暖色调下划线，不压正文阅读节奏）。这是本主题关键词标记的唯一方式。
- **行内代码**：暖纸底色 `background:#f7f0e6;color:#b8532a;padding:1px 6px;border-radius:3px;font-family:monospace;font-size:13px`。不要用冷灰色底——破坏暖纸氛围。
- **删除线**：`text-decoration:line-through;color:#b8a898`，仅用于表达「否定/修改」语义。
- 约束：同一段落行内代码不超过 2 处；下划线标记每屏不超过 2 处。

### 5. 引用/金句块

**设计方向**：大引号文艺款为主，辅以杂志侧栏式。

- **变体 A — 大引号装饰（默认，文艺气质）**：这是暖纸墨的灵魂组件。大号陶土色引号 `"`（`font-size:36px;color:#b8532a;line-height:0.6`）置于引用文字上方，文字 `font-size:15px;color:#4a3728;font-weight:bold;line-height:1.85`。无底色、无边框、无竖条——就引号+文字。像翻阅杂志时被标记出来的金句。
- **变体 B — 左侧竖条（经典克制）**：`border-left:3px solid #b8532a;padding:6px 0 6px 12px;background:transparent`。用于严肃引用或长段引用（超过 40 字）。
- **变体 C — 暖底卡片（强调包裹）**：`background:#f7f0e6;padding:14px 16px;border-radius:4px`。用于核心结论，需要与正文有明确视觉区隔时使用。
- 选用原则：金句/观点用 A；长者言论/书籍引用用 B；核心结论用 C。
- 约束：同一文章中大引号（变体 A）不超过 3 处；变体 A 的引号必须是陶土色 `#b8532a`（不可用其他色）。
- **金句提取策略**：暖纸墨的金句提取标准不同于极简蓝——更重视「情感浓度」和「可引用性」：
  - 包含比喻、类比、意象的句子（如「像...一样」「仿佛」）
  - 结尾带有反问或感叹的句子
  - 段落首句或尾句中节奏感强的短句（10-20 字最佳）
  - 原文中已有引号包裹的内容
  提取后，情感金句用变体 A（大引号），逻辑结论用变体 B（竖条），论证核心用变体 C（暖底卡）。

### 6. 提示/旁注块

**设计方向**：细线框 + 暖底，像杂志侧栏。

- 结构：`border:1px solid #e8d5c4;background:#f7f0e6;padding:10px 14px;border-radius:4px`。
- 顶部可选小标签：用小号 9px 陶土色 `#b8532a` + letter-spacing:2px 标记「旁注 / NOTE」。
- 文字：`font-size:13px;color:#8b7355;line-height:1.75`。
- 变体方向：可用左侧 3px 陶土色竖条替代四周细线框（更轻盈）；可去除顶部标签仅保留框+文字。
- 约束：提示块内不超过 3 行；同一屏不超过 1 个。

### 7. 数据/步骤展示

**设计方向**：暖底色卡片 + 陶土编号，像手工制作的索引卡。

- **数据展示**：三列 flex 卡片，暖纸底 `#f7f0e6`。大数字用 `#b8532a` bold 28px，标签用 `#b8a898` 10px。可在卡片之间插入 1px `#e8d5c4` 竖线作为分隔。
- **步骤（横向）**：圆形陶土编号（`width:28px;height:28px;background:#b8532a;color:#fff;border-radius:50%`）+ 步骤名。注意：暖纸墨的编号背景可使用镂空（`background:transparent;border:2px solid #b8532a;color:#b8532a`）替代实心圆，减轻重量感。
- **步骤（纵向）**：编号左对齐，步骤间用 `border-left:1px solid #e8d5c4` 连接（形成时间轴效果）。
- **表格数据**：flex 模拟行，偶数行暖纸底 `#fdfaf4` 上叠加浅底 `#f7f0e6`，表头陶土色文字。
- 约束：卡片圆角 4px；编号保持圆形（实心或镂空二选一但全文统一）。
- **数据可视化决策树**：
  - 单一核心数据 → 居中大字展示（`font-size:36px;color:#b8532a;font-weight:bold`），下方加细横线
  - 2-3 个对比数据 → 三列暖底卡片 + 列间 1px 竖线
  - 4+ 数据行 → 表格（flex 模拟，偶数行暖底 `#f7f0e6`）
  - 流程/步骤 → 纵向步骤流 + 竖线连接（时间轴效果）
  - 时间线 → 纵向步骤流，步骤间用 `border-left` 细线连接

### 8. 图片与媒体

**设计方向**：照片式处理——极细边框 + 微弱阴影，像相册里的照片。

- 标准图片：暖纸底色容器 + `border:1px solid #e8d5c4`。可加极微弱的阴影 `box-shadow:rgba(0,0,0,0.06) 1px 1px 6px 0px`——暖纸墨是唯一可接受微妙阴影的主题（模拟照片浮在纸面上的感觉）。
- 圆角策略：图片容器 `border-radius:4px`，与卡片统一。
- 图片说明：`font-size:11px;color:#b8a898;text-align:center;margin-top:6px;font-style:italic`（斜体说明更有杂志编辑感）。
- GIF 角标：暖纸底 `#f7f0e6` + 陶土色文字 `#b8532a` 胶囊标签。
- 约束：阴影色必须极淡（rgba 透明度 ≤0.08）；不使用虚线边框。

### 9. 代码块

**设计方向**：深棕暗底——像老式印刷厂的铅字盘，而非科技终端。

- 深棕底色 `#3d3226`（不是纯黑，是深棕色），顶栏 `#2c241b`。
- 代码文字 `#e8d5c4`（米色文字在深棕底上，比纯白柔和），等宽字体栈。
- 可选：使用浅色代码块——暖纸底 `#f7f0e6` + 左侧 3px 陶土色竖条，代码文字 `#4a3728`。浅色版更适合短代码片段。
- 行内代码：暖纸底 `background:#f7f0e6;color:#b8532a`。
- 约束：暖纸墨的主题气质倾向于浅色代码块（保持纸张感），深棕代码块用于较长代码段。没有硬性规定，AI 根据代码长度和上下文选择。

### 10. 分割线

**设计方向**：细线型——1px 细横线，居中短线段。

- 样式 A（章节级大分隔）：1px `#e8d5c4` 全宽横线，`margin:32px 0`。
- 样式 B（小节级小分隔）：1px `#e8d5c4` 居中短线段（宽 60% 或 120px），`margin:24px auto`。
- 样式 C（装饰型）：`· · ·` 暖棕色 `#b8a898`，letter-spacing:6px，`margin:24px 0`。
- 变体方向：可在细线中央嵌入一个小菱形 `◆` 或小圆点 `·`（陶土色），但必须小且不抢戏。
- 约束：分割线必须是水平横线（不竖线也不斜线）；宽度 1px；颜色固定 `#e8d5c4` 或 `#b8a898`。

### 11. 尾部互动卡

**设计方向**：暖纸明信片风格——像夹在杂志里的回函卡。

- 卡片底色：暖纸 `#f7f0e6`，上边框 `1px solid #e8d5c4`，下边框同。
- 引导文案：`font-size:13px;color:#8b7355;text-align:center`。示例：「感谢你读到这里」「如果有所共鸣」——有温度但不矫情。
- 互动按钮：推荐风格 B 胶囊渐变——`linear-gradient(135deg,#b8532a,#c97a4a)` 暖色胶囊。或使用风格 A 经典按钮——纯陶土色底 `#b8532a`。不推荐极简文字链（太冷不适合暖纸调性）。
- 可选：在互动按钮上方加一短行装饰文字（如「— THE END —」），`font-size:10px;color:#b8a898;letter-spacing:3px`。
- 结束语：`font-size:10px;color:#b8a898;font-style:italic;text-align:center`。像杂志末页的编辑寄语。
- 约束：尾部卡片高度不超过 200px；最多 3 个互动按钮；细线框颜色固定 `#e8d5c4`。

## 文章类型到设计策略

| 文章类型 | 设计策略 | 节奏特点 | 重点组件 |
|---------|---------|---------|---------|
| 技术教程 | 减少暖纸氛围（少用大引号），增加步骤流清晰度；细横线标记章节切换 | 均匀节奏，步骤间细线分隔 | 步骤流（纵向）、行内代码、提示块、表格数据 |
| 工具测评 | 数据卡片暖底+陶土数字；图片占比高（工具截图）；标签镂空式 | 信息密度中，数据卡穿插 | 数据卡片、图片容器、标签（镂空）、分割线 B |
| 知识整理 | 章节标题主导；提示块用细线框侧栏式 | 舒缓，章间大分隔线 | 章节标题、提示块、引用块 B、分割线 A |
| 观点分析 | **这是暖纸墨的黄金场景**。大引号提炼每段核心观点；段落节奏有张有弛；首段斜体导语定调 | 慢起→观点密集→收束，像一篇好的杂志文章 | 大引号（变体 A）、导语段、引用块 A、分割线 C |
| 深度文章 | 组合使用大引号+竖条引用；长段落保持阅读沉浸感；每屏仅 1 个陶土锚点 | 极慢节奏，像翻阅长文杂志 | 大引号、引用块 B、章节标题、分割线 A |
| 人文思考 | **暖纸墨的天然主场**。大量使用大引号；正文深棕有温度；图片黑白或暖色调；留白不规则 | 散文节奏，留白服务于情绪 | 大引号（变体 A，可多次使用）、图片容器、首段导语、细横线分割 |
| 产品深潜 | 数据卡+暖底代码块组合；陶土色仅用于关键数字；章节标题清晰导航 | 信息密度高，数据与文字交替 | 数据卡片、代码块、章节标题、步骤流 |

## Markdown 到设计决策指南

| Markdown 元素 | 设计决策 | 自由度说明 |
|--------------|---------|-----------|
| `# 一级标题` | 转为文章主标题，`font-size:20-22px;color:#4a3728;font-weight:bold;text-align:center` | 字号微调；可选上方加细横线装饰 |
| `## 二级标题` | 顶部细横线+编号+标题，`font-size:18px;color:#4a3728` | 编号格式（数字/中文/罗马）；细线在上或在下 |
| `### 三级标题` | 无线的纯编号标题，`font-size:16px;color:#4a3728` | 可加左侧陶土色小圆点装饰 |
| 正文段落 | `<p>` 标签，14px `#4a3728`，行高 1.85，两端对齐 | 段落间距 8-12px 弹性调整 |
| `**加粗**` | `<strong>` 直接包裹，颜色不变（`#4a3728`） | 不用彩色加粗 |
| `*斜体*` | `<em>` 包裹，导语推荐斜体 `color:#8b7355` | 正文斜体谨慎使用 |
| `` `行内代码` `` | 暖纸底 `#f7f0e6` + 陶土色 `#b8532a`，等宽字体 13px | 圆角 3-4px |
| `> 引用` | 短金句（<25 字）默认用变体 A 大引号；长者言论用变体 B 竖条；核心结论用变体 C 暖底 | AI 判断引用长度和性质选择 |
| ` ``` 代码块 ``` ` | 短代码用浅色暖底版；长代码用深棕暗底 `#3d3226` | 深/浅选择由 AI 根据代码长度和上下文决定 |
| `- 列表 / 1. 有序列表` | 纵向步骤流（时间轴式竖线连接）或镂空编号列表 | 空心/实心编号可微调 |
| `--- 分割线` | 细横线（1px `#e8d5c4`）或暖棕 `· · ·` | 章节级用全宽实线，小节级用短实线或装饰点 |
| `![图片](url)` | 暖底容器 + 细边框 + 极微阴影 `rgba(0,0,0,0.06)` | 阴影可去掉；圆角 4-6px |
| `[链接](url)` | 转为暖色下划线 `border-bottom:2px solid #e8c4a2` | 下划线颜色固定为 `#e8c4a2` |
| 表格 | flex 模拟行，偶数行 `#f7f0e6` 底色，表头陶土色字 | 表头可用暖底替代白底 |

## 平台铁律

- 所有中文文字必须用 `<span leaf="">` 包裹
- 禁用 `<div>/<style>/<script>/class/id/position:absolute/float/@media/grid/var()`
- 每个元素必须携带 `box-sizing:border-box;` 和 `max-width:100%!important;`
- `<p>` 标签必须 `margin:0;padding:0;box-sizing:border-box;`
- 所有 CSS 必须内联
- 最大宽度 677px，正文字号 14px（不可改）

## 快速气质校验清单

排版完成后，AI 应自查以下问题：

- [ ] 全文底色是否为暖纸色 `#fdfaf4`（非纯白）？
- [ ] 细横线（hairline）是否贯穿全文作为核心视觉语言？是否 1px `#e8d5c4`？
- [ ] 陶土色 `#b8532a` 是否仅出现在锚点位置（大引号/编号/竖条/按钮）？每屏不超过 2 处？
- [ ] 正文和标题是否统一使用深棕 `#4a3728`（而非纯黑）？
- [ ] 引用金句是否优先使用了大引号（变体 A）？
- [ ] 标签是否使用了镂空式（边框+陶土字）而非实心药丸？
- [ ] 卡片底色是否为暖纸浅底 `#f7f0e6`？
- [ ] 代码块底色是否为深棕 `#3d3226` 或暖纸浅底 `#f7f0e6`（而非冷灰黑）？
- [ ] 图片是否使用了极微弱阴影（可选）？阴影透明度 ≤0.08？
- [ ] 所有中文文字是否 `<span leaf="">` 包裹？

## 常见设计陷阱与避免模式

以下是 AI 在使用暖纸墨主题进行排版时最常犯的错误，请逐条对照检查：

**陷阱 1：默认白底遗忘**——忘记设置全文底色 `#fdfaf4`，使用了纯白 `#ffffff`。暖纸墨的灵魂是纸张色——纯白底会丢失所有温度。

**陷阱 2：纯黑文字**——标题或正文使用了 `#000` 或 `#1a1a1a` 而不是深棕 `#4a3728`。纯黑在暖纸上显得生硬刺眼，像打印错误。

**陷阱 3：实心药丸标签**——使用了蓝色实心或陶土实心标签。暖纸墨的标签应该是镂空的（`border:1px solid #e8d5c4;color:#b8532a`），保持轻盈通透。

**陷阱 4：大引号滥用**——每屏出现多个大引号 `"`。大引号是暖纸墨最强烈的视觉标记，全文不超过 3 处。想象杂志排版——只有最精彩的句子才值得放大和标引号。

**陷阱 5：细横线（hairline）缺席**——全文没有出现任何细横线。细横线是暖纸墨的结构骨架——至少在章节标题处应该出现。如果没有细横线，文章在视觉上就没有杂志的「翻阅感」。

**陷阱 6：冷灰色入侵**——使用了 `#f5f5f5`、`#e5e5e5` 等冷灰色作为卡片底色或边框。暖纸墨的所有灰色必须带暖色调——`#f7f0e6`（暖底）、`#e8d5c4`（暖边框）、`#b8a898`（暖注释色）。

**陷阱 7：代码块颜色错位**——使用了 `#1e293b`（极简蓝的代码底色）或 `#1a1a2e`（暗夜青的文章底色）。暖纸墨的代码块要么是深棕 `#3d3226`（暗色），要么是暖纸浅底 `#f7f0e6`（浅色）。不要让冷蓝/冷黑出现在暖纸文章中。

**陷阱 8：尾部过于冷淡**——使用了极简文字链风格（无按钮）。暖纸墨是有温度的——互动按钮应该使用暖色胶囊渐变或陶土色实心按钮，给读者温暖的告别。

## 排版决策流程速查

AI 拿到一篇 Markdown 文章后，按以下决策链进行排版设计：

```
1. 扫描文章，判断文章类型 → 查「文章类型到设计策略」表确定节奏和重点组件
2. 设置全文底色 → #fdfaf4（暖纸色），这是第一步，不可省略
3. 扫描文章，提取金句 → 按情感浓度选变体（情感金句→A 大引号 / 逻辑结论→B 竖条 / 论证核心→C 暖底卡）
4. 扫描文章，识别数据点 → 按「数据可视化决策树」选展示形式
5. 生成封面信息卡 → 根据文章气质选方向（观点→目录式 / 人文→引语式 / 深度→纯标题式）
6. 遍历章节生成章节标题 → 顶部细横线+陶土编号+标题，全文统一
7. 遍历段落生成正文 → 14px 深棕 #4a3728，行高 1.85，注意首段斜体导语
8. 段落间穿插细横线 → 每章结束时一条全宽细横线（hairline = 1px #e8d5c4）
9. 遇到代码块 → 短代码暖纸浅底 / 长代码深棕暗底 #3d3226
10. 遇到图片 → 暖底容器+细边框+极微阴影（可选）
11. 生成分割线 → 细横线或 · · ·（暖棕色）
12. 生成尾部互动卡 → 暖色胶囊按钮或陶土实心按钮 + 暖纸明信片框
13. 全文自检 → 对照「快速气质校验清单」
```
