# 通用组件设计参考 —— 代码块 · 图片 · 分割线

> **跨所有主题通用**。主题专属组件（封面卡、章节标题、引用块、信息卡、签名区等）在各主题设计语言文件中定义；本文件提供三类所有主题共用的组件**设计参考**。
>
> **使用方式**：以下是设计参考而非固定模板。AI 在排版时参考这些组件的结构和设计原则，根据当前主题的设计变量（主色/底色/文字色）进行适配，**每次排版可以有不同的细节演绎**。
>
> **平台铁律**（所有变体必须遵守）：
> - 禁 `<style>/<script>/<div>/class/id/position:absolute/float/@media/grid/var()`
> - 所有 CSS 内联，所有中文文字 `<span leaf="">` 包裹
> - `box-sizing:border-box;` + `max-width:100%!important;` 每个元素必带

## 一、代码块

代码块内的英文、半角符号、缩进原样保留（代码不适用"中文全角标点"规则）。

### 设计原则

- **必须用代码块组件**，不要塞进普通段落或引用块
- **每行一个 `<p style="margin:0">`**，绝不用 `white-space:pre`（会把源码缩进渲染成大空白）
- 缩进用全角空格 `　`，不用源码空格
- 行距靠 `line-height:1.6`，保持紧凑

### 三种变体

**深色终端（默认，技术感强）**：
- 底色 `#1E293B`，顶栏 `#0F172A` + 三色圆点
- 代码色 `#E2E8F0`，语言标签色 `#64748B`
- 等宽字体栈：`'SF Mono',Consolas,Monaco,monospace`
- 适合所有主题，尤其暗色系

**浅色代码块**：
- 底色 `#F6F8FA`，左边框 3px 主题主色
- 顶栏浅底 + 语言标签
- 代码色 `#24292F`
- 适合浅色温暖主题（极简蓝/暖纸墨/森语绿）

**行内代码**：
- 浅灰底 `#F1F5F9` + 主题主色文字
- `padding:1px 6px;border-radius:4px`
- 字号与正文同（14px），等宽字体

### 设计自由度

- 深/浅选择由 AI 根据主题气质决定（暗色主题默认深色，浅色主题可选浅色）
- 顶栏语言标签可选（无语言可省略）
- 圆角/阴影可微调，但保持克制

## 二、图片与媒体

### 设计原则

- `<img>` 用 `max-width:100%;height:auto;display:block;margin:0 auto`——按图片自身尺寸居中，大图缩到容器宽，小图保持原尺寸。**不用 `width:100%`** 强制铺满
- 有 alt 说明才加说明行（灰色小字居中），空 alt 不编造说明
- 图片容器圆角/阴影可微调，但保持克制

### 三种变体

**标准图片**：
- 白底容器 + 浅灰边框 + 微阴影
- 有说明时：图片下方灰色居中说明文字
- 无说明时：仅图片容器

**GIF 动图**：
- 与标准图片相同，但加"GIF 动图"角标（主题浅底 + 主题深字色小药丸标签）
- 无色块极简主题可改用细线描边胶囊

**待补素材占位（居中板块）**：
- 柔虚线框 + 居中图标（🎬视频/🖼图片/📊信息图/📎附件）+ 说明
- 这是**唯一允许虚线框的场景**——表达"占位/待补"语义
- 底色柔和不抢戏，各主题可微调虚线色/底色为当前主题中性色

### 设计自由度

- 图片容器圆角值可微调（4px~12px 范围）
- GIF 角标形状可微调（药丸/圆角矩形）
- 待补板块图标可根据素材类型选择

## 三、分割线

### 设计原则

- 分割线的气质必须与主题一致——它是主题性格的延伸
- 不要太重（不要抢内容的风头）
- 上下间距 24px 起，给足呼吸空间
- 纯装饰，不带语义

### 变体方向（按主题气质选择）

**文字装饰型**（适合极简/编辑风）：
- 灰色 `···` 或 `· · ·`，letter-spacing 6-8px
- 适合：极简蓝、绯红编

**细线型**（适合温暖/杂志风）：
- 细横线，主题浅色，居中短线段
- 适合：暖纸墨、森语绿

**符号装饰型**（适合科技/奢华风）：
- `◆ ◆ ◆` 或 `— ◆ —`，主题主色
- 适合：暗夜青、墨金雅

### 设计自由度

- 分割线样式完全由 AI 根据主题气质选择
- 可以创造新的分割线形态（点阵/渐变线/符号组合），但保持轻盈不抢戏
- 同一篇文章内分割线风格统一

## 四、数据展示组件

### 设计原则

- 数据是文章的信息锚点，需要视觉突出但不过度设计
- 保持主题色板约束
- 移动端友好（三列数据在窄屏仍可读）

### 变体方向

**三列数据卡**：
- flex 横向排列，每列等宽
- 大数字（28px+）+ 小标签（10px）
- 可用主题浅底做卡片背景，也可纯白+边框
- 适合：数据报告、工具对比

**横向步骤流**：
- flex 横向排列，圆形编号 + 步骤名 + 描述
- 编号用主题主色实心圆
- 适合：教程、操作指南

**纵向步骤流**：
- flex 纵向排列，圆形编号左对齐 + 右侧标题描述
- 步骤间用浅色分割或留白区分
- 适合：详细教程、方法论

**表格（Markdown 表格转换）**：
- 用 flex 模拟，不用 `<table>` 标签
- 偶数行浅灰底，表头用主题主色
- 适合：参数对比、数据罗列

### 设计自由度

- 数据卡片的圆角、阴影深浅可微调
- 步骤编号可用纯数字/药丸/圆形，根据主题气质选择
- 表格可简化为卡片列表（信息密度低时）

## 五、提示与标签

### 设计原则

- 提示块用于"划重点"，不是正文本身
- 左竖条 > 四周框（虚线框笨重）
- 标签徽章轻量、不抢戏
- 类型标记（提示/注意/重点/Prompt/旁注）可灵活选用

### 变体方向

**左竖条提示块**：
- 左侧 3-4px 主题主色竖条 + 浅底
- 可选顶部小标签（提示/注意/重点）
- 适合：旁注、补充说明

**标签徽章组**：
- 横向 flex wrap，主题主色背景 + 白字
- 圆角小药丸，字号 9-10px
- 适合：关键词标签、分类标记

### 设计自由度

- 提示块是否有顶部小标签由 AI 决定
- 标签徽章的圆角/间距可微调
- 可以创造新的提示形态（如左侧 icon + 文字），但保持简洁
