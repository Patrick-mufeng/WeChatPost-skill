# 绯红编 -- 设计语言定义

> 本文件是**绯红编**主题的设计语言定义。它为 AI 提供一套完整的视觉决策框架，使 AI 能够理解"绯红编"的编辑风骨，并据此为每篇文章进行原创排版设计。本文件中的组件描述是**灵感参考**而非复制粘贴模板 -- AI 应在理解设计原则后自主演绎。


## 一、设计变量速查表

```
主色：       #c1292e（绯红 -- 克制使用：章节编号、关键锚点 ≤5 处、左边框。像报纸的红标题，少即是权威）
辅色：       #a32025（深绯红 -- 用于二级强调、数据展示中次要数字的强调）
浅底：       #fef9f7（极浅暖红底 -- 引用块/提示块背景，像被红墨水轻微浸染的纸）
浅边框：     #e8d5d0（暖灰粉边框 -- 卡片描边，有温度的灰）
强调点缀：   #c1292e（同主色 -- 高亮文字背景的极浅版 rgba(193,41,46,0.08)）
标题色：     #1a1210（接近墨色的深棕 -- 标题，比纯黑更有温度）
正文色：     #2a2218（深灰棕接近墨色 -- 正文，模仿报纸铅字）
辅助文字：   #6e6058（中暖灰 -- 次要信息、说明文字）
注释色：     #9e9088（浅暖灰 -- 日期、来源、最小文字层级）
分割线色：   #d8ccc4（暖灰 -- 细线分割，中间可点缀红色菱形）
正文字号：   14px（不可改）
正文行高：   1.85
段落间距：   8px（标准间距，结构感来自层级对比而非留白加大）
章节标题字号：18-22px
章节间距：   32px 上
字间距：     0.3px
最大宽度：   677px
字体栈：     -apple-system,BlinkMacSystemFont,'PingFang SC','Microsoft YaHei',sans-serif
页面底色：   #fefcf7（微暖白 -- 像略微泛黄的新闻纸，比纯白柔和）
正文关键词下划线 CSS：border-bottom:2px solid #f4c2c2;font-weight:600;
```


## 二、设计原则与气质

### 气质关键词
**编辑风骨** **红白张力** **结构感** **克制权威** **报刊基因**

### 配色哲学

这是一套"红笔批注"色彩系统。页面底色 #fefcf7 是新闻纸的白 -- 不是冷白，是带着纸浆温度的暖白。正文色 #2a2218 是墨色 -- 不是纯黑，是铅字压进纸面的深棕。红色是编辑手中的笔：只在章节编号、关键锚点（全文 ≤5 处）、左边框出现。红色的权威感来自它的稀少 -- 如果到处是红色，就没有重点了。灰阶系统承担信息层级：从标题的深棕到注释的浅暖灰，四个灰度构成清晰的信息等级。色彩逻辑是"一份被精心编辑过的报纸"：黑白是底色，红色是编辑的批注。

**色彩分配比例**：页面底色 80% / 浅底 8% / 文字灰阶 10% / 主色 ≤2%。红色面积占比极低但视觉权重极高 -- 每个红色出现的位置都是读者视线的"编辑推荐位"。

### 视觉节奏

- **段落节奏**：标准 8px 段落间距。节奏不靠加大间距，而靠层级之间的结构跳跃 -- 从正文到章节标题的断裂感、从正文到引用块的红色左边框突然打断。段落内部可以用 `<br/>` 制造微停顿（如作者换气的时刻）。
- **焦点密度**：每 2-3 段可有一个标记点（加粗/下划线/锚点）。但主色锚点全文 ≤5 处，确保红色的每次出现都有"翻到重点版"的心理暗示。如果文章很长（>5000 字），可放宽到 ≤7 处。
- **留白策略**：留白服务于结构而非氛围。章节上下 32px，组件上下 20px。不用"多出来的空白"制造气氛，用"精确的间距"建立秩序。每个间距数值都有结构意义 -- 不是随机的"多一些"。
- **阅读感**：像读一份精心排版的周报 -- 信息密度中高，但层级清晰，视线可以快速定位到想读的部分。红色的出现像"编辑推荐"的标记。

### 强弱规则

| 层级 | 定义 | 频率限制 | 色彩角色 |
|------|------|---------|---------|
| 锚点层 | 章节编号、全文核心金句锚点、数据中的关键数字 | 全文 ≤5 处主色出现 | 主色 #c1292e |
| 标记层 | 行内加粗、下划线关键词、引用左竖条 | 每段 ≤2 处标记 | 标题色加粗 / 主色下划线 / 主色竖条 |
| 容器层 | 引用块、提示块、信息卡、数据卡片 | 按需使用，不限制频率 | 浅底 #fef9f7 + 浅边框 |

**核心原则**：红色是权威的编辑标记。每次红色出现都应该让读者感到"这里是编辑认为最重要的部分"。红色出现 5 次以内，这个效果成立。红色出现 20 次，效果坍塌。

### 设计禁区

- 不要在主色背景上放浅色文字做卡片（红色底+白字 = 警示牌，不是编辑标记）
- 不要在同一个组件中同时使用主色和辅色（二选一）
- 不要使用纯黑 #000 -- 所有"黑"色都用深棕 #1a1210 或墨色 #2a2218
- 不要使用过大的圆角（>8px）-- 编辑风要方正、有理性的边角
- 不要使用阴影来制造层次 -- 用边框和底色差异来区分层次


## 三、组件设计模式

### 1. 封面/头部信息卡

**设计方向**：**报头档案式** -- 像报纸头版的信息摘要栏。

**设计原则**：用一条红色细线（2px，主色 #c1292e）作为卡片的顶部边框，宽度 100%（铺满卡片宽度），下方 padding: 16px 16px。标签用主色背景小药丸 -- 字号 9px，白色文字，font-weight: bold，letter-spacing: 2px，padding: 3px 10px，圆角 2px -- 最多 3-5 个横向排列。字数和阅读时间用辅助文字色 10px，放在标签行下方，用"|"分隔（不是"·" -- "|"更有编辑感）。概括句用标题色 14px weight 600，作为独立的摘要行 -- 像报纸的副标题。

**变体方向**：
- **标准报头式**：顶部红横线 + 标签 + 元信息 + 概括句
- **侧边竖条式**：左侧 3px 主色竖条 + 右侧内容区（标签/元信息/概括句），更紧凑，适合标签较少的文章

**使用场景判断**：
- 深度评测/行业分析：必用，标签偏学术/领域关键词
- 案例复盘：必用，标签偏行业/方法论关键词
- 观点文章：可选（如果文章直接以观点开场可省略）

**约束边界**：红色只出现在顶部边框或左侧竖条（二选一）。标签可以用主色药丸，但不超过 5 个。不要用阴影。卡片 padding 不小于 16px。

### 2. 章节标题

**设计方向**：**编号+左竖条** -- 编辑风的经典章节标识。

**设计原则**：左侧 3px 主色竖条（高度与标题文字等高），竖条与标题间距 8px。上方一个微型标签（字号 9px，主色，letter-spacing: 2px，weight: bold）标注章节编号如 `SECTION 01`。标题用标题色 18-22px weight 600。单行。竖条+标签+标题形成明确的三段式层级。

**变体方向**：
- **标准章节**：微型标签（`SECTION 01`）+ 竖条 + 标题。标签在竖条左侧或上方。
- **中文编号式**：微型标签改为中文如 `第一章`，适合文学性较强的文章。
- **简化式**：仅竖条 + 标题（省略微型标签），用于子章节。竖条可降为 2px。
- **尾声章**：竖条变为分割线色 #d8ccc4（灰化处理），表达"编辑注脚"的语义 -- 读者看到灰色竖条就知道文章接近尾声了。

**约束边界**：竖条必须用主色（标准章）或灰化版（尾声章）。微型标签必须大写英文字母间距（letter-spacing ≥ 2px），或中文小字间距。竖条高度 ≦ 标题行高。

### 3. 正文段落

**基准样式**：14px / 1.85 行高 / 0.3px 字间距 / 正文色 #2a2218 / margin: 0 0 8px 0 / text-align: justify。

**首段处理**：首段字号不变，但首行之前加一个微型引导标签（字号 9px，主色，letter-spacing: 2px，weight: bold），内容为 `导读` 或 `EDITOR'S NOTE`，像一个编辑在正文开始前做的批注。该标签后紧跟正文（间距 4px）。AI 可根据文章气质决定是否使用 -- 深度报道和长文推荐使用，短评和轻量文章可省略。

**段落对齐**：正文两端对齐。引用和说明文字左对齐。诗歌和极短过渡句可以居中。

**段落间节奏控制**：连续的短段落（1-2 行）可以制造紧张感。长段落（>5 行）后应接一个短段落或视觉停顿元素来释放压力。

### 4. 行内标记

- **加粗**：使用 `<strong>` 标签，颜色使用标题色 #1a1210（比正文更深），weight 700。加粗在绯红编中是"加黑印刷"：同一色系但更重，不是换成红色。每段最多 2 处加粗。
- **高亮/标记**：使用 `background: rgba(193,41,46,0.08)`（主色的 8% 透明度），padding: 1px 3px，无圆角。像一个用红笔轻划的背景标记。全文使用不超过 5 处。
- **下划线**：关键词下划线使用 `border-bottom: 2px solid #f4c2c2`（柔和的浅红下划线），font-weight: 600。下划线是红色在行内标记中的唯一出现方式，且只用浅红（#f4c2c2）而非主色 -- 保持克制。
- **行内代码**：浅暖灰底 #f5f0ed + 标题色文字，等宽字体，圆角 3px，padding: 1px 5px。
- **删除线**：注释色 #9e9088，line-through。
- **斜体**：中文不使用斜体。英文斜体保留 `<em>` + 正文色。

**约束**：全文不使用主色 #c1292e 做行内文字颜色（颜色值）。主色只属于锚点层和边框层。下划线用浅红而非主色 -- 这是绯红编的一个关键差异点。

### 5. 引用/金句块

绯红编的引用块有三种变体方向，从左到右"编辑干预度"递增。

**变体 A -- 左竖条引用（视觉重量：中。默认，编辑风）**：
左侧 3px 主色竖条 + 浅底 #fef9f7 + padding: 12px 14px。文字字号 14px，正文色，行高 1.8。竖条是"编辑标注"的视觉语言。适合引用他人言论、资料引用、案例中的关键数据摘录。可在引用下方用注释色 10px 标注出处。

**变体 B -- 红底金句（视觉重量：重。核心观点专用）**：
浅底 #fef9f7 + 左上方微型红色引导标签（`金句` 或 `KEY INSIGHT`，字号 8px，主色，letter-spacing: 1.5px，weight: bold）+ 文字字号 15px weight 600 标题色，行高 1.85。适合全文最重要的 1-2 句核心观点。全文最多使用 2 次。

**变体 C -- 报纸引号式（视觉重量：轻。文艺引用）**：
左侧大号引号（字号 40px，主色 20% 透明度 -- rgba(193,41,46,0.2)），引号与文字在同行（inline-block 排列）。文字 14px 辅助文字色。适合在严肃分析中引入一段文学性引用，制造质感反差。全文 1-2 次。

**选用指南**：变体 A 用于资料引用（常规，不限次数）；变体 B 用于核心观点（全文 ≤2 次）；变体 C 用于文学性引用（点缀，1-2 次）。

**反模式**：不要对同一段话同时使用变体 B 的标签和下划线的红色强调（红色叠加 = 噪音）。不要在引用块内使用加粗 -- 引用块自己就是"被标记"的内容。

### 6. 提示/旁注块

**设计方向**：**编辑批注式** -- 像一个编辑在空白处写的旁注。

**设计原则**：左侧 3px 辅色 #a32025 竖条 + 浅底 #fef9f7 + 左上角微型标签（`旁注` 或 `NOTE`，字号 8px，辅色，letter-spacing: 1.5px）。文字使用辅助文字色 #6e6058，字号 13px，行高 1.75。padding: 12px 14px。

**使用场景**：
- **补充信息**：正文放不下的背景信息 -- 用"旁注"标签
- **编辑小记**：AI/编辑对正文某段的评论 -- 用"NOTE"标签
- **注意事项**：温和的提醒 -- 用"注意"标签（辅色，不是主色）

**约束边界**：竖条用辅色而非主色（区分于引用块 -- 引用是"重要的"，旁注是"额外的"）。微型标签可选 -- 如果内容明显是提示性质且上下文已经提示了，省略标签更干净。全文提示块不超过 3 个（太多旁注说明正文写得不够好）。

### 7. 数据/步骤展示

**设计方向**：**报纸信息图式** -- 数据展示像报纸中的统计图板块。

**设计原则**：
- **数据卡（三列）**：三列卡片用白底 + 浅边框 #e8d5d0 + 顶部 2px 主色线。关键数字用标题色字号 28px weight 700（不用主色 -- 主色留给标题线）。标签用注释色 10px。卡片 padding: 14px 10px，gap: 12px。
- **数据卡（单列大数字）**：单个大数字 + 上方主色微型标签 + 下方注释色说明。适合突出的单个数据。
- **步骤展示**：纵向布局，每步用左对齐布局 -- 步骤编号用主色实心圆形（直径 24px，白字 bold 11px）+ 右侧步骤标题 weight 600 标题色 + 描述辅助文字色 13px。步骤之间间距 12px。
- **表格转换**：偶数行用极浅底 #fdfaf8（比页面底稍暖），表头行用浅底 #fef9f7 + 底部 1px 主色线。使用 flex 模拟，不用 `<table>`。

**约束边界**：数据数字不用主色 -- 主色只用于装饰线/编号圆。表头底线是主色在数据区的唯一出现方式。步骤编号的圆形直径统一 24px。

### 8. 图片与媒体

**设计原则**：
- **图片容器**：白底 + 浅边框 #e8d5d0 + 8px 圆角 + 可选极轻阴影（rgba(0,0,0,0.04) 1px 1px 6px -- 这个阴影在大部分屏幕上几乎不可见，只是让图片与底色之间有一个微妙的"抬离"感）。
- **图片说明**：注释色 #9e9088，11px，居中。说明前加一个微型菱形 `◆`（主色 30% 透明度 -- rgba(193,41,46,0.3)）作为编辑标记。说明与图片间距 6px。
- **GIF 角标**：用辅色背景 + 白色文字（字号 8px weight bold，padding: 2px 6px，圆角 2px）的标签，放在图片右上角或独立放在图片下方说明行前面。
- **待补素材**：虚线框用浅边框色 #e8d5d0（1px dashed），底色 #fefcf7。中央 emoji + 注释色说明。

**约束边界**：阴影极轻（几乎看不见）。圆角 4-12px 范围。GIF 角标用辅色而非主色（辅色是"功能标记"，主色是"编辑标记"）。

### 9. 代码块

**设计方向**：**深色终端（首选）** -- 与浅色新闻纸形成黑白反差，编辑部档案中夹了一份技术文档。

**设计原则**：深底 #1e2228 + 顶栏 #15181c + 三色圆点（红 #ff5f57 / 黄 #febc2e / 绿 #28c840）+ 语言标签（注释色 9px）。代码文字 #d0d4dc，字号 12px，行高 1.7。每行代码用独立的 `<p>`（margin: 0），缩进用全角空格。

**变体**：浅色代码块（底 #f5f2ef + 左主色 3px 竖条，无顶栏）作为备选，适合代码行数少于 10 行的场景 -- 此时深色终端的大面积黑块太突兀。

**选用判断**：
- 代码行数 > 10 行：用深色终端
- 代码行数 ≤ 10 行：可用浅色变体
- 文章中有 3 个以上代码块：深色和浅色交替使用制造节奏

**约束边界**：深色代码块在绯红编中是"反差元素"，如果文章超过 3 个深色代码块，考虑将部分转为浅色变体。

### 10. 分割线

**设计方向**：**细线 + 红色菱形** -- 像报纸中不同报道之间的分隔符。

**设计原则**：一条 1px solid 分割线色 #d8ccc4 的细横线，居中，宽度 80px，上下间距 24px。在横线正中央叠加一个红色微型菱形 `◆`（主色 #c1292e，字号 8px），菱形与横线同层显示。

**变体**：
- `— ◆ —`：左右短横 + 中央红菱（略复杂，适合大章节分隔）
- `· ◆ ·`：左右点 + 中央红菱（更轻盈）
- 纯细线（无菱形）：用于子章节之间的轻分隔

**约束边界**：菱形必须小（≤8px），不可用大号符号。分割线不可铺满全宽 -- 60-100px 居中。文章内最多使用 1 次带菱形分割线（主色出现次数有限）。

### 11. 尾部互动卡

**设计方向**：**编辑室告别卡** -- 像杂志最后一页的编辑部信息栏。

**设计原则**：顶部一条 2px 主色横线（铺满 100%），下方 padding: 20px 16px。引导文案用标题色 15px weight 600 居中 -- 如"如果你读到了这里"、"这篇文章值得一个在看"、"分享给需要的人"。下方间隔 10px 放置三个按钮：
- **点赞**：主色背景 #c1292e + 白色文字，padding: 8px 20px，圆角 2px（近乎直角 -- 编辑风不圆润）
- **在看**：辅色背景 #a32025 + 白色文字，padding: 8px 20px，圆角 2px
- **转发**：白底 + 1px 浅边框 #e8d5d0 + 标题色文字，padding: 8px 20px，圆角 2px

底部告别语用注释色 11px 居中 -- 如"我们下篇文章见"。

**互动按钮设计**：使用风格 A（经典三连按钮 -- 实心色块长方形）。不用胶囊圆角（border-radius > 4px），不用渐变。

**告别语文案方向**：
- 深度分析/评测：专业但不冷 -- "感谢你的阅读时间"
- 案例复盘：务实 -- "希望对你有帮助"
- 观点文章：有力 -- "不一样的看法，值得被看到"

**约束边界**：顶部红线铺满 100% 宽度。按钮用长方形（圆角 2px）而非胶囊。本文案不走"跪求"路线 -- 绯红编的权威感不支持这种语气。


## 四、文章类型 -> 设计策略

| 文章类型 | 设计策略 | 节奏特点 | 重点组件 |
|---------|---------|---------|---------|
| 教程/方法 | 用深色代码块承载代码/命令，步骤用主色圆形编号。信息层级清晰，每个步骤有明确视觉标记。方法论的"为什么"用变体 A 引用块。 | 快节奏。步骤紧密排列（间距 12px），代码块是节奏停顿点。步骤-代码-步骤形成三拍子节奏。 | 深色代码块、圆形步骤编号、提示块、左竖条引用 |
| 盘点/清单 | 每项用浅底 + 顶部微红线（1px 主色）的卡片承载。项目间用纯细线分隔（无菱形）。编号可用主色文字。 | 像翻目录 -- 每项一个独立、可扫描的单元。读者可以跳着看。每项标题足够有吸引力。 | 浅底卡片、细线分割、标签徽章、主色编号 |
| 观点/议论 | 核心观点用变体 B（红底金句）突出。论据段落用正文 + 加粗关键短语。反对观点用变体 C（报纸引号）展示并标注。 | 观点块是"重音"，正文是"行进"。观点出现的位置控制全文节奏。建议每 4-6 段正文插一个观点锚点。 | 红底金句、左竖条引用、报纸引号、行内加粗 |
| 访谈/对话 | 问句用标题色 weight 600 + 左侧微型主色菱形标记 `◆`（字号 6px）。答句用正文。人物切换时用分割线（纯细线，无菱形）。 | 问句是鼓点，答句是旋律。菱形标记让问句在扫描时一眼可见。长答句适当拆段。 | 菱形标记、左竖条、行内加粗、纯细线分割 |
| 数据/报告 | 亮出"报纸信息图"风格：三列数据卡 + 顶部红线。单列大数字用主色微型标签引导。图说文字用注释色。 | 数据卡是视觉锚点集群。正文围绕数据展开。数据卡放置节奏：每 2-3 段正文插入一组数据。 | 数据展示（三列+单列）、分割线（带菱形）、表格转换 |
| 随笔/叙事 | 绯红编的随笔比森语绿更"编辑过"：有结构、有标记、有明显的编辑意图。用变体 C（报纸引号）制造文学质感。章节标题可以简化。 | 节奏介于编辑风和文学风之间。保留结构感但不压迫。全文红色出现降到 ≤3 处。 | 报纸引号式引用、行内下划线、简化章节标题、首段引导标签 |
| 案例复盘 | 绯红编的主场。用左竖条引用承载案例中的关键数据/引述。章节标题用标准编辑风，每个案例独立成章。关键发现用变体 B（红底金句）。 | 章节感最强。每章的编号+竖条是扫描的锚点。案例背景->分析->发现的三段式结构对应三种视觉节奏。 | 章节标题、左竖条引用、红底金句、数据展示、提示块 |

### 文章类型识别与设计决策流程

1. **判断文章的结构密度**：高结构（教程/案例/盘点）-- 多用章节标题、步骤编号、数据卡。低结构（随笔/叙事）-- 简化装饰，红色出现降低到 ≤3 处
2. **判断红色配额分配**：全文红色 ≤5 处。优先分配给章节标题（必须），其次分配给核心观点（变体 B），再其次分配给数据区的装饰线
3. **判断代码块策略**：技术类文章代码块是主角 -- 用深色终端。非技术类文章代码块是陪衬 -- 用浅色变体
4. **混合型文章**：按章节独立处理。技术章用深色代码+步骤流，分析章用红底金句+左竖条引用


## 五、Markdown -> 设计决策指南

| Markdown 元素 | 设计决策 | 自由度说明 |
|--------------|---------|-----------|
| `# 标题`（一级） | 文章主标题：标题色 #1a1210，字号 22-24px，weight 700，居中或左对齐。可选下方一条 40px 宽主色细线作为标题底线。 | AI 可选择加/不加底线（底线消耗 1 次红配额）。字号 ±2px。对齐方式：分析类左对齐，故事类居中。 |
| `## 标题`（二级） | 章节标题：左侧 3px 主色竖条 + 微型标签（`SECTION 01`）+ 标题，18-22px weight 600。 | 微型标签可用英文/中文。竖条高度与标题等高。 |
| `### 标题`（三级） | 子章节：仅竖条 + 标题（省略微型标签），16-18px weight 600。竖条可降为 2px 辅色。 | 竖条粗细和颜色可微调。 |
| `**加粗**` | `<strong>` 标签，标题色 #1a1210 weight 700。每段 ≤2 处。 | 无自由度。加粗是加黑，不换色。 |
| `*斜体*` | `<em>` 标签，正文色。中文不使用斜体。 | AI 优先用加粗或下划线替代中文斜体。英文斜体可保留。 |
| `` `行内代码` `` | 浅暖灰底 #f5f0ed + 标题色等宽字，圆角 3px。 | 底色/圆角可微调。 |
| ` ```代码块``` ` | 深色终端（默认）：深底 #1e2228 + 三色圆点顶栏。代码行数 ≤10 可选浅色变体。 | 深/浅选择由行数和文章调性决定。 |
| `> 引用` | 变体 A（左竖条）/ B（红底金句）/ C（报纸引号）三选一。 | AI 根据引用性质：资料用 A，核心观点用 B，文学引用用 C。 |
| `- 列表` | 圆点使用主色 30% 透明度 rgba(193,41,46,0.3)。列表项间距 6px。 | 圆点透明度可微调（20%-40%）。 |
| `1. 有序列表` | 序号使用主色 #c1292e，weight 600。 | 无自由度。 |
| `![图片](url)` | 白底 + 浅边框 + 8px 圆角 + 极轻阴影。说明文字前加微型菱形 `◆`。 | 圆角范围 4-12px。菱形可选（消耗红配额）。 |
| `---` 分割线 | 80px 宽细线 + 中央红色菱形 `◆`，上下 24px。子章节之间纯细线无菱形。 | 菱形消耗红配额。宽度 60-100px 可调。菱形变体可选（`— ◆ —` 等）。 |
| `[链接](url)` | 转为行内文字标记，使用浅红下划线（#f4c2c2，非主色）。 | AI 决定是否保留链接文字并标注来源。 |
| `| 表格 |` | 转为 flex 模拟表格。偶数行极浅底，表头行底部 1px 主色线。禁止 `<table>`。 | 可转为卡片列表（信息密度低时）。表头底线消耗 1 次红配额。 |

**平台铁律**：所有含中文的文本节点必须使用 `<span leaf="">文字</span>` 包裹。禁止 `<style>`、`<script>`、`<div>`、`position: absolute/fixed/relative`。所有元素必须带 `box-sizing: border-box;` 和 `max-width: 100% !important;`。
