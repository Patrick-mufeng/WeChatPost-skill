## 选择建议

- **用户选择制 + 智能推荐**：用户没指定主题时，据文章题材推荐最契合的 1 个（标"推荐"放第一位）+ 2 个备选，列出全部让用户一步确认。
- **题材→主题契合参考**：
  - 技术教程 / 工具测评 / 知识整理 → 极简蓝（默认）
  - 观点分析 / 深度思考 / 人文 → 暖纸墨
  - 开发技术 / 数据报告 / 产品深潜 → 暗夜青
  - 随笔 / 生活方式 / 个人反思 → 森语绿
  - 深度评测 / 行业分析 / 案例复盘 → 绯红编
  - 人物访谈 / 品牌叙事 / 高端内容 → 墨金雅
- **全自动模式**（用户说"直接排 / 一键 / 不用问"）：自动选默认主题（极简蓝），交付时说明选择理由。
- **C 选项**：用户选 C（AI 自主设计）→ 不套任何预设主题，AI 根据文章内容自主定义色彩/气质/节奏，产出独一无二的视觉方案。
- **同一篇文章只用一套设计语言**，不混搭。

正文关键词下划线一律用上表"正文关键词下划线 CSS"列的值。这是该主题下划线标记的单一权威来源。

## 新主题登记流程

1. 创建设计语言文件 `references/theme-{英文标识}.md`（格式要求：五章节结构——设计变量 / 设计原则与气质 / 组件设计模式 / 文章类型→设计策略 / Markdown→设计决策指南）。
2. 在上表登记一行；首个/最常用主题放第一行作为默认推荐。
3. 确保所有设计变量符合公众号平台约束（见 `design-spec.md`），颜色对比度 ≥ 4.5:1。
