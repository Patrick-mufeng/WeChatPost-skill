# 墨金雅 -- 设计语言定义

> 本文件是**墨金雅**主题的设计语言定义。它为 AI 提供一套完整的视觉决策框架，使 AI 能够理解"墨金雅"的经典气质，并据此为每篇文章进行原创排版设计。本文件中的组件描述是**灵感参考**而非复制粘贴模板 -- AI 应在理解设计原则后自主演绎。

## 二、设计原则与气质

### 气质关键词
**墨色** **金饰** **经典比例** **厚重** **收藏感**

### 配色哲学

这是一套"暗房中的金箔"色彩系统。页面底色 #1a1a1a 是精装书的硬壳内页 -- 不是死黑，是带着纸浆温度的极深灰。正文 #d4d0c8 是暖灰白，不是冷白 -- 有温度，像在暖色灯光下阅读。金色 #c9a96e 是这本书唯一被允许的奢侈：只在章节编号、关键引号、极少高亮处出现。全文金色出现 ≤5 处 -- 每出现一次，都像书页间夹了一片金箔。灰阶系统在暗色中承担微妙的层次递进：从标题暖灰白到注释深灰，四个亮度层级在暗底上制造"摸得到的层次"。卡片底色 #252525 与页面底色 #1a1a1a 的亮度差极小 -- 层次来自微妙的明度递进而非颜色对比。

**色彩分配比例**：页面底色 80% / 卡片浅底 10% / 文字暖灰白 8% / 金色点缀 ≤2%。金色在墨金雅中的角色不是"颜色"，而是"材质" -- 它像烫金工艺一样出现在书的封面和扉页。

**关于"不用纯黑 #000"和"不用纯白 #fff"**：墨金雅的所有"黑色"区域使用 #1a1a1a 而非 #000，所有"白色"文字使用 #d4d0c8 而非 #fff。这是因为纯黑和纯白在暗色阅读中对比度过高（> 12:1），长时间阅读会产生视觉疲劳。暖灰白 + 极深灰的组合对比度约 9:1 -- 清晰但舒适。

### 视觉节奏

- **段落节奏**：段落间距 10px + 行高 1.9，文字在暗色中"漂浮"得更多。长段落需要更多行间距来保持可读性 -- 如果段落超过 6 行，考虑用 `<br/>` 或拆段来制造呼吸点。每 3-4 段插入一个视觉停顿（金句引用/图片/分割线）。
- **焦点密度**：由于暗色背景自然聚焦注意力，每屏最多 1 个视觉焦点（金色锚点或引用块）。金色出现位置即阅读高潮。在金色出现之前的 2-3 段正文应该"蓄力" -- 不加粗、不标记、让读者沉浸在文字中，然后金色像聚光灯一样打在最关键的那句话上。
- **留白策略**：留白在暗色背景中比在浅色中"更重" -- 同样的 36px 间距在暗底上感觉比在亮底上更空。因此墨金雅的留白策略更克制：章节上下 36px（不用森语绿的 40px），组件间隔 20px。留白是"经典比例"的结果而非侘寂的表达。
- **阅读感**：像在私人图书馆的台灯下翻一本限量精装书 -- 专注、沉浸、有仪式感。金色偶尔闪过的瞬间是这本书给你的"彩蛋"。

### 强弱规则

| 层级 | 定义 | 频率限制 | 色彩角色 |
|------|------|---------|---------|
| 锚点层 | 章节编号数字、核心金句的大引号、极少数行内高亮 | 每章 1 处，全文金色 ≤5 处 | 点缀色 #c9a96e |
| 标记层 | 行内加粗、关键词金色下划线、引用左金色装饰 | 每段 ≤1 处加粗，下划线 ≤3 处 | 标题色加粗 / 金色下划线 |
| 容器层 | 卡片底色、引用块、信息卡 | 按需使用 | 浅底 #252525 |

**核心原则**：金色是这本书的"金边"。如果每页都有金边，金边就失去了意义。金色出现的位置必须是经过"编辑选择"的全书最重要的 5 个瞬间。

### 设计禁区

- 不要在非暗色背景上使用这套色彩系统（墨金雅的暗底是基因，不是可选参数）
- 不要使用金色做大面积背景 -- 金色只在"点"和"线"层级出现（文字/下划线/编号），不做"面"
- 不要使用纯白 #fff 或纯黑 #000
- 不要使用鲜艳的颜色（任何饱和度 > 30% 的非金色系颜色都会在暗底上"跳"得太厉害）
- 不要使用阴影 -- 暗色背景中阴影不可见
- 不要使用过大的圆角（>6px）-- 精装书是方正的


## 三、组件设计模式

### 1. 封面/头部信息卡

**设计方向**：**档案卡片式** -- 像精装书扉页的版权信息栏，但经过现代设计处理。

**设计原则**：浅底 #252525 卡片 + 左侧 2px 金色竖线（点缀色 #c9a96e）。卡片 padding: 20px 16px。标签用小号金色文字（字号 9px，点缀色，letter-spacing: 2px），不用药丸 -- 金色文字本身就足够显眼，不需要再加背景色。字数和阅读时间用辅助文字色 #9e9a90，字号 10px，放在一行用"·"分隔。概括句用标题色 14px weight 500，像扉页上那句"献给某某"的题词。

**变体方向**：
- **标准档案式**：左侧金竖线 + 浅底卡片 + 金色标签文字 + 元信息 + 概括句
- **居中题词式**：无卡片底板，概括句居中 + 上下各一条 40px 宽金色装饰细线。适合文章标题本身就足够强、不需要元信息的场景

**使用场景判断**：
- 人物访谈/品牌叙事/高端内容：必用档案式
- 深度特稿/长文：必用（元信息帮助读者做阅读预期）
- 元信息极少（< 2 个标签）：可改用居中题词式

**约束边界**：金色在头部信息卡中仅出现 1-2 次（竖线+标签文字计为 1 次金色出现）。标签不用金色药丸背景。不用阴影。

### 2. 章节标题

**设计方向**：**金色编号 + 留白** -- 编号本身就是全部装饰。

**设计原则**：章节编号使用点缀色 #c9a96e，字号 20-22px，weight 400（不加粗 -- 金色本身就足够了）。编号独占一行，下方标题用标题色 18-20px weight 500。编号和标题之间 4px 间距。章节整体上方 36px 间距，下方 8px。不使用任何装饰条/竖线/背景色块 -- 金色数字在暗色页面上已经足够夺目。

**变体方向**：
- **标准数字式**：`01`（金色）独占一行 + 标题（暖灰白）在下一行
- **中文编号式**：`第一章`（金色）+ 标题，适用文学性较强的文章
- **罗马数字式**：`I.` 或 `II.`（金色），适合极正式/经典感文章
- **尾声章**：编号使用注释色 #6e6a62（金被灰替代），表达"终章的沉静" -- 金色在最后一步退场

**约束边界**：编号不用圆形/药丸包裹。不用左边框。编号不加粗。金色的出现仅靠"编号"这一个载体。子章节（`###`）不使用金色编号 -- 那是二级标题的特权。

### 3. 正文段落

**基准样式**：14px / 1.9 行高 / 0.5px 字间距 / 正文色 #d4d0c8 / margin: 0 0 10px 0 / text-align: justify。

**字符间距说明**：字间距 0.5px 是墨金雅在排版层面的关键差异化参数。暗色背景下中文字符需要更多呼吸空间来维持辨识度。这不是"宽松"，是"经典排版比例"的自然结果 -- 铅字印刷时代的字间距本身就比现代屏幕排版大。

**首段处理**：首段字号可升至 15px，颜色使用标题色，weight 400。像精装书正文的第一个段落 -- 比后续段落稍大，但不喧哗。这是建议而非规则。如果文章以一段有力的引文开头，首段加大特别合适。

**段落对齐**：正文两端对齐。引用和诗歌左对齐。独句过渡段可居中。

**暗色背景下的长段落处理**：暗色中阅读长段落更容易疲劳。如果段落超过 6 行（约 200 字），考虑在自然断句处拆分。拆分后的第一个新段落用一个 `<br/>` 而非完整的新 `<p>` -- 保持内容的连续性同时提供视觉停顿。

### 4. 行内标记

- **加粗**：使用 `<strong>` 标签，颜色保持正文色 #d4d0c8，weight 600。墨金雅中的加粗是"铅字加印"，不换色。每段 ≤1 处加粗。
- **高亮/标记**：使用 `background: rgba(201,169,110,0.12)`（金色的 12% 透明度），padding: 1px 2px，无圆角。像一个在暗色页面上用极淡金色墨水划过的痕迹。全文金色高亮 ≤3 处，且计入金色总配额。
- **下划线**：关键词下划线使用 `border-bottom: 2px solid #c9a96e`（金色下划线），font-weight: 600。这是金色在行内标记中的核心出现方式 -- 替代加粗做重点标记。全文金色下划线 ≤3 处，且计入金色总配额。
- **行内代码**：深灰底 #222225 + 正文色文字，等宽字体，圆角 3px，padding: 1px 5px。暗色代码在暗色页面中靠深灰底与页面底色 #1a1a1a 的微弱差异区分。
- **删除线**：注释色 #6e6a62，line-through。
- **斜体**：中文不使用斜体。英文斜体保留 `<em>` + 正文色。

**金色配额管理**：金色高亮 + 金色下划线 合计全文 ≤3 处。如果一篇文章已经有章节编号（约 3-5 章 = 3-5 次金色出现），行内金色标记应该更少。AI 必须在排版前先"预算"全文的金色出现次数。

### 5. 引用/金句块

墨金雅的引用块有三种变体方向，按"金色使用量"和"视觉重量"排序。

**变体 A -- 居中金句（视觉重量：重。首选，经典感最强）**：
文字居中，字号 16px，标题色，weight 500，行高 1.9。文字上方有一个金色左引号（字号 40px，点缀色，line-height: 0.6，像悬挂在句子上方的金饰）。上下各 20px 留白。适合全文最重要的 1-2 句金句。金色引号 + 居中文字 = 这本书的"金句页"。**消耗 1 次金色配额。**

**变体 B -- 暗色卡片引用（视觉重量：中。资料引用）**：
浅底 #252525 卡片 + 左侧 2px 金色竖线 + padding: 14px 16px。文字字号 14px，正文色，行高 1.85。适合引用他人言论或外部资料。可在引用下方用注释色 10px 标注出处。**消耗 1 次金色配额（竖线）。**

**变体 C -- 金色小标引用（视觉重量：中轻。要点提取）**：
浅底 #252525 卡片 + 顶部金色微型标签（`摘录` 或 `EXCERPT`，字号 8px，点缀色，letter-spacing: 2px）+ 正文 14px。适合从正文中提取的"要点"，不是金句但值得标记。**消耗 1 次金色配额（标签文字）。**

**选用指南**：变体 A 用于全文最核心的金句（≤2 次）；变体 B 用于外部资料引用；变体 C 用于要点提取（可多用，但注意金色配额）。

**反模式**：不要在变体 A 中放入超过 25 字的内容 -- 长居中金句会失去"金句感"。不要在同一篇文章中对同一引用同时使用变体 A 的金引号和变体 C 的金标签（金色叠金 = 噪音）。

### 6. 提示/旁注块

**设计方向**：**暗色卡片 + 顶部小标**。

**设计原则**：浅底 #252525 卡片 + padding: 14px 16px + 顶部金色微型标签（`旁注` 或 `NOTE`，字号 8px，点缀色，letter-spacing: 2px）。文字使用辅助文字色 #9e9a90，字号 13px，行高 1.75。**消耗 1 次金色配额。**

**无金变体**：省略金色标签，仅用浅底卡片 + 辅助文字色文字。适合不需要金色标记的普通提示，不消耗金色配额。

**使用场景**：
- **需要金色标记**：编辑的重要补充、"请注意"级别的提示（全文 ≤2 次）
- **不需要金色标记**：技术说明、背景信息补充（不限次数）

**约束边界**：金色标签版提示块全文 ≤2 次（因为消耗金色配额）。无金版提示块不限。不用左边框 -- 暗色卡片本身已经有足够边界感。

### 7. 数据/步骤展示

**设计方向**：**暗色经典比例卡片**。

**设计原则**：
- **数据卡（三列）**：三列卡片用浅底 #252525 背景 + 可选极暗边框 #333333。关键数字用标题色字号 28px weight 600（不用金色 -- 金太珍贵）。标签用注释色 10px。卡片 padding: 14px 10px，gap: 12px。
- **数据卡（单列大数字）**：单个大数字 + 上方金色微型标签（可选，消耗配额）+ 下方注释色说明。适合作文的"关键数据亮点"。
- **步骤展示**：纵向布局，编号用金色文字（01/02/03，字号 14px，点缀色 -- 步骤编号是金色在章节编号之外的第二合理出现场景）。步骤标题用标题色 weight 600，描述用辅助文字色 13px。每步间距 14px。**金色配额处理**：如果步骤编号使用金色，一组步骤（如 3 步）计为 1 次金色配额消费（因为是同一个"视觉组"）。
- **表格转换**：偶数行用浅底 #252525，奇数行用页面底 #1a1a1a（保持背景一致性）。表头行用浅底 + 底部 1px 金色线。使用 flex 模拟，不用 `<table>`。**表头金线消耗 1 次金色配额。**

**约束边界**：数据数字不用金色。金色在数据区只出现在：步骤编号组（计 1 次）、表头底线（计 1 次）、可选单列数据卡标签（计 1 次）。如果金色配额已用完，所有金色降为标题色/注释色。

### 8. 图片与媒体

**设计原则**：
- **图片容器**：无额外背景（暗色页面中使用透明或继承底色 -- 白色图片容器在暗底中像一块"补丁"，除非图片本身有白底需要承载）。圆角 4px。如果图片有白底，给它一个微妙的圆角让白底边缘柔化。
- **图片说明**：注释色 #6e6a62，11px，居中。说明前可选金色微型菱形 `◆`（字号 6px）-- 如果金色配额未用完。
- **GIF 角标**：用浅底 #252525 + 正文色文字的轻量标签（字号 8px，padding: 2px 6px，圆角 2px）。不用金色 -- GIF 标记不配消耗金色配额。
- **待补素材**：虚线框用分割线色 #3a3a3a（1px dashed），底色 #1a1a1a（与页面底色同），注释色占位文字。

**约束边界**：图片容器不加白底分区（除非图片自带了白底需要圆角修饰）。不加阴影。圆角 ≤6px。金色菱形消耗配额。

### 9. 代码块

**设计方向**：**深色终端** -- 与墨金雅暗色底自然融合，且需要"更深"来建立层级。

**设计原则**：深底 #0d0d0f（比页面底色 #1a1a1a 更深 -- 代码块是页面中的一块"纯黑档案"），顶栏 #16161a + 三色圆点。代码文字 #c8ccd0。语言标签用注释色 9px。代码行高 1.6，字号 12px。每行独立的 `<p>`（margin: 0），缩进用全角空格。

**变体**：无顶栏版本 -- 仅深底 #0d0d0f + padding: 16px，更简洁。适合不超过 10 行的短代码。

**约束边界**：代码块底色必须明显深于页面底色 #1a1a1a（深度差是唯一的层次手段）。不用浅色代码块 -- 浅色代码块在暗色页面中像"闪光弹"。代码块在墨金雅中不使用任何金色装饰（金不属于代码区）。

### 10. 分割线

**设计方向**：**暗线 + 可选金色符号**。

**设计原则**：一条 1px solid 分割线色 #3a3a3a 的暗横线，居中，宽度 80px，上下间距 24px。分割线在暗色背景中几乎不可见 -- 这正是目的：它是一个"触感"而非"视觉"，像书页间的纹理变化，读者几乎注意不到但潜意识里感知到分隔。

**金色变体**：如果该分割线位置值得用金色（如文章从上半部分过渡到下半部分、主要章节之间的重大切换），可在横线中央加一个金色微型菱形 `◆`（点缀色，字号 8px）。**消耗 1 次金色配额。**

**变体**：`— ◆ —`（金色菱形 + 左右短横），适合全书级别的部分切换（如 Part 1 到 Part 2）。

**约束边界**：金色菱形分割线全文 ≤1 次（因为金色配额有限且分割线不是内容）。其余分割线用纯暗线。不用点线、波浪线或任何花哨分割线。

### 11. 尾部互动卡

**设计方向**：**精装书后记式** -- 像限量书最后一页的出版信息栏。

**设计原则**：浅底 #252525 卡片 + padding: 24px 16px。顶部一句引导文案：标题色 15px weight 500 居中 -- 如"合上这本书之前"、"感谢你读到了最后"、"下篇文章再会"。下方间隔 12px 放置三个按钮：
- **点赞**：金色背景 #c9a96e + 墨黑文字 #1e1f23，padding: 8px 20px，圆角 2px（近乎直角）。**消耗 1 次金色配额。**
- **在看**：浅底 #3a3c40 + 标题色文字，padding: 8px 20px，圆角 2px
- **转发**：透明底 + 1px 浅边框 #333333 + 标题色文字，padding: 8px 20px，圆角 2px

底部告别语用注释色 11px 居中。

**互动按钮设计**：使用风格 E（暗色标签卡 -- 方形微圆角按钮）或者自定义。关键是"方正"而非"圆润"。金色按钮是墨金雅尾部卡的核心识别元素。

**告别语文案方向**：
- 人物访谈/品牌叙事："感谢你花时间了解这个故事"
- 深度特稿："每一篇长文都是一次对话。谢谢你参与。"
- 高端内容："下期见。"

**约束边界**：金色按钮（点赞）是尾部唯一一处金色。不用渐变。按钮圆角 ≤4px。告别语不走热情路线也不走冷淡路线 -- 墨金雅的告别是"有分量的感谢"。


## 四、文章类型 -> 设计策略

| 文章类型 | 设计策略 | 节奏特点 | 重点组件 |
|---------|---------|---------|---------|
| 教程/方法 | 深色代码块是主角，步骤编号用金色文字（计 1 次金配额）。正文精简，让代码和步骤承担主要信息传递。方法论的"为什么"用无金提示块。 | 快节奏。代码块之间用短正文过渡。金色步骤编号是阅读路径的导航点。每 3-4 步插入一个代码块作为实践节点。 | 深色代码块、金色步骤编号、无金提示块 |
| 盘点/清单 | 每项用浅底 #252525 卡片承载。项与项之间用纯暗线分割（不消耗金）。编号可选金色文字（消耗配额）。每项标题 weight 600。 | 沉稳的翻阅感。每项卡片在暗色页面中像一个个独立的档案盒。读者可以跳读。 | 暗色卡片、暗线分割、金色编号（可选） |
| 观点/议论 | 核心观点提取为居中金句（变体 A），金引号 + 居中文字。论据用正文 + 金色下划线标记关键短语。反对观点用变体 B（暗卡+金竖线）。 | 金句是"静止时刻" -- 读者停下来读那句话。正文是"行进"。金句出现前 2-3 段正文"蓄力"不加任何标记。 | 居中金句、金色下划线、暗色卡片引用 |
| 访谈/对话 | 人物姓名用金色文字（合理消耗金配额 -- 访谈中这是金的最佳使用场景之一）。回答用正文。切换人物时用暗线分割。 | 金色人名是"谁在说话"的标记，在暗底上像聚光灯打在说话者的名字上。问答节奏清晰。 | 金色文字标记、暗色卡片、暗线分割 |
| 数据/报告 | 用暗色三列数据卡。数据数字用标题色 28px（不用金），标签注释色。表头底部可以用金色底线标记重要数据区。 | 数据卡是暗色中相对"亮"的区域（#252525 vs #1a1a1a）。每个数据卡是一个视觉锚点。数据密集的文章适当增加分割线。 | 数据展示（三列+单列）、暗色卡片、金色表头线（可选） |
| 随笔/叙事 | 墨金雅的随笔偏向"深沉"。首段加大字号 + 0.5px 字间距。全文金色只出现在章节编号和 1 处居中金句。段落长而沉稳。 | 慢、沉、有重量。像深夜独白。金色偶尔闪过像窗外的灯火。全文 2-3 次金色出现即止。 | 居中金句（≤1 次）、首段加大字号、暗线分割、极简章节标题 |
| 案例复盘 | 每个案例用暗色卡片包裹，案例标题用标题色 weight 600。关键发现用变体 C（金色小标引用）标记。每个案例独立成完整的暗卡单元。 | 结构清晰但不过度。暗色卡片取代章节标题成为案例的分界标记。案例间用暗线分割。 | 暗色卡片、金色小标引用、步骤编号、数据展示 |

### 文章类型识别与设计决策流程

1. **判断金色配额预算**：先数章节数量（每章编号 = 1 次金色）。剩余配额分配给行内标记、引用、数据装饰。优先级：章节编号 > 居中金句 > 金色下划线 > 步骤编号 > 表头金线 > 分割线金菱
2. **判断暗色卡片使用密度**：高结构文章（教程/案例）-- 卡片密集（每章 1-3 个）。低结构文章（随笔/叙事）-- 卡片稀疏（全文 2-3 个）
3. **判断代码块策略**：墨金雅只有深色终端一个选项。关键是代码块与页面底色的深度差 -- 代码块底色必须比页面底色深
4. **混合型文章**：按章节独立处理。金配额在全文范围内统一管理，不按章独立


## 五、Markdown -> 设计决策指南

| Markdown 元素 | 设计决策 | 自由度说明 |
|--------------|---------|-----------|
| `# 标题`（一级） | 文章主标题：标题色 #d4d0c8，字号 22-24px，weight 600，居中。可选下方一条 40px 宽金色细线（消耗 1 次金配额）。 | 金线可选。字号 ±2px。 |
| `## 标题`（二级） | 章节标题：金色编号独占一行（20-22px，点缀色）+ 标题下一行（18-20px，标题色 weight 500）。上间距 36px。 | 编号格式可选（`01`/`一`/`第一章`/`I.`）。 |
| `### 标题`（三级） | 子章节：标题色 16px weight 500，上间距 28px。无金色编号。 | 可选左金色微型圆点（消耗配额）。 |
| `**加粗**` | `<strong>` 标签，正文色 weight 600。每段 ≤1 处。 | 无自由度。加粗是加印，不换色。 |
| `*斜体*` | `<em>` 标签，正文色。中文不使用斜体。 | AI 优先用加粗或金色下划线替代中文斜体。 |
| `` `行内代码` `` | 深灰底 #222225 + 正文色等宽字，圆角 3px。 | 底色深度可微调。 |
| ` ```代码块``` ` | 深色终端：深底 #0d0d0f + 三色圆点顶栏，代码色 #c8ccd0。 | 有/无顶栏可选。短代码（≤10行）推荐无顶栏。 |
| `> 引用` | 变体 A（居中金句+金引号）/ B（暗卡+金竖线）/ C（暗卡+金小标）三选一。 | AI 根据引用性质选择。各变体消耗金配额（1 次/个）。 |
| `- 列表` | 圆点使用辅助文字色 #9e9a90。列表项间距 6px。 | 圆点可选金色（消耗配额）。 |
| `1. 有序列表` | 序号使用辅助文字色，或者金色（消耗配额）。 | AI 决定是否消耗金配额。 |
| `![图片](url)` | 无背景容器 + 4px 圆角。说明前可选金色菱形 `◆`（消耗配额）。 | 圆角范围 2-6px。金色菱形可选。 |
| `---` 分割线 | 80px 暗线（+ 可选金色菱形），上下 24px。 | 金色菱形消耗配额（全文 ≤1 次）。大部分分割线用纯暗线。 |
| `[链接](url)` | 转为行内文字标记，使用金色下划线（消耗配额）或普通辅助色文字标记。 | AI 决定重要链接是否用金色下划线。 |
| `| 表格 |` | 转为 flex 模拟表格。偶数行浅底 #252525，表头底部可选金线。禁止 `<table>`。 | 金线可选（消耗配额）。可转为暗色卡片列表。 |

**平台铁律**：所有含中文的文本节点必须使用 `<span leaf="">文字</span>` 包裹。禁止 `<style>`、`<script>`、`<div>`、`position: absolute/fixed/relative`。所有元素必须带 `box-sizing: border-box;` 和 `max-width: 100% !important;`。
