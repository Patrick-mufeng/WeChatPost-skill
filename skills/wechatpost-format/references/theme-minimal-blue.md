# 极简蓝 设计语言

> 本文件是「极简蓝」主题的**设计语言定义**，非组件模板。目标是让 AI 理解这套设计语言后，为每篇文章进行**原创排版设计**。下文组件描述为设计原则与灵感参考，AI 应根据文章内容自由演绎，而非逐字复制。

## 设计原则与气质

### 气质关键词

克制、理性、呼吸感、秩序、效率

### 配色哲学

极简蓝的核心配色逻辑是「蓝色做锚点，灰色做支撑，白色做呼吸」。主色 `#2563eb` 不出现在大面积区域中，仅用于需要读者「定位」的瞬间——章节编号、左侧竖条、按钮、关键行内链接。灰阶体系（`#333` / `#666` / `#999`）承担 90% 的信息层级表达。留白面积占页面 40% 以上，让每个视觉元素有自己的「领地」。

蓝色不是装饰，是信号。读者在扫读时，蓝色标记的位置就是「这里值得停一下」——这种信息导航功能是极简蓝设计语言的本质。

具体到色彩分配比例：全文 95% 以上的像素是灰阶（白/浅灰/中灰/深灰），蓝色占比不超过 3%，其余 2% 为深色代码块区域。这种极端的不对称分配正是「克制」二字的量化表达。

### 视觉节奏

- **段落间距**：段落间 margin 8-10px，正文行高 1.85，确保单行 14px 文字在 677px 宽度下约 35-38 个汉字，符合最优阅读宽度。
- **焦点密度**：每屏（约 600-800px 高度）最多 1-2 个蓝色锚点，其他区域以灰色和留白填充。读者的注意力是有配额的，不要在同一屏放置超过两个蓝色元素。
- **留白策略**：章节之间 32px 上间距；组件之间 20-24px；标题与正文之间 12px。留白本身就是一种「标点」——告诉读者一个信息单元结束了。
- **节奏控制**：正文段落后跟 8px 间距 = 正常呼吸；20px 间距 = 话题转换；32px 间距 = 章节切换。

### 强弱规则

本主题引入三层信息标记体系：

| 层级 | 名称 | 视觉表现 | 频率限制 |
|------|------|---------|---------|
| 锚点层 | 蓝色标记 | 章节编号、左侧 3px 竖条、关键链接下划线、按钮底色 | 每屏 1-2 处 |
| 标记层 | 加粗/浅底 | 正文加粗、卡片浅蓝底 `#f5f7fa`、行内浅灰底代码 | 每段 ≤2 处加粗 |
| 容器层 | 灰边框/白底 | 卡片白底+灰边框、分割线、图片容器 | 无限制 |

原则：锚点层像路标（要少要精准），标记层像高亮笔（要克制），容器层像纸张（自然而然）。

## 组件设计模式

> 以下为设计指导，非固定模板。AI 应根据具体文章特点，在这些约束边界内自由创造。

### 1. 封面/头部信息卡

**设计方向**：档案式信息卡——像一张工整的技术档案封面。

- 浅蓝底 `#f5f7fa` 整块卡片，不铺满（留左右 padding）。
- 顶部：3-5 个话题标签，使用蓝色小药丸 `background:#2563eb;color:#fff;font-size:9px`，或灰色边框标签 `border:1px solid #e2e8f0;color:#666`。标签数量 ≤5 个。
- 中部：文章标题，`font-size:20-22px`，深色 `#1a1a2e`，`font-weight:bold`。
- 底部一行：字数 + 阅读时间 + 一句话概括。元数据用 `#999` 10px，概括句用 `#666` 12px。
- 变体方向：可去除蓝色底色改为纯白+浅边框；可在左侧加一条 3px 蓝色竖条做侧标记。
- 约束：信息卡高度不超过一屏的 1/3；不使用渐变、不使用阴影（保持理性感）。
- **封面设计自由度**：除了档案式，可尝试如下方向：
  - **极简标题式**：无卡片包裹，纯白底+标题+一行元数据。适合极短文章。
  - **标签云式**：标题下用 3-5 个蓝色镂空标签排列，制造视觉锚点群。
  - **引导句式**：用一句 15px 斜体导语代替概括句，更有人文气息。
  AI 应根据文章气质选方向——技术教程用档案式，知识整理用极简标题式，工具测评用标签云式。

### 2. 章节标题

**设计方向**：左侧蓝色竖条 + 编号 + 标题。

- 结构：`3px 蓝色竖条 | 英文编号（如 SECTION 01）| 中文标题`。
- 竖条 `width:3px;height:20px;background:#2563eb;flex-shrink:0`。
- 编号用 `font-size:10px;color:#2563eb;letter-spacing:2px;font-weight:bold`。
- 标题用 `font-size:18px;color:#1a1a2e;font-weight:bold`。
- 变体方向：纯数字编号（01/02）替代 SECTION 前缀；无竖条仅用蓝色编号+标题；用浅蓝色短线 `#93c5fd` 替代深蓝竖条（更轻柔）。
- 约束：每个章节标题上方 margin 32px，下方 12px；编号始终存在（帮助扫读定位）。

### 3. 正文段落

**设计方向**：纯文字，不装饰，靠节奏说话。

- 基准：`font-size:14px;color:#333;line-height:1.85;letter-spacing:0.3px;text-align:justify`。
- 首段处理：不做首字下沉。极简蓝主张「所有段落平等」，首段与其他段落样式一致。可选在首段前加一句导语（`font-size:15px;color:#666`，斜体或用竖条标记）。
- 段落间距：`margin:8px 0`，连续段落保持统一节奏。
- 约束：每段不超过 5-6 行（约 200 字），超过则拆分；段落间禁止使用装饰性符号。

### 4. 行内标记

**设计方向**：蓝灰二元标记系统，拒绝五彩斑斓。

- **加粗**：`<strong>` 直接使用，颜色保持正文 `#333`，不加额外颜色。加粗是「加重语气」而非「指向链接」。
- **高亮/下划线**：使用 `border-bottom:2px solid #93c5fd;font-weight:600`（浅蓝色下划线，不压正文阅读节奏）。这是本主题关键词标记的唯一方式。
- **行内代码**：浅灰底 `background:#f1f5f9;color:#2563eb;padding:1px 6px;border-radius:4px;font-family:monospace;font-size:13px`。
- **删除线**：`text-decoration:line-through;color:#999`，仅用于表达「否定/废弃」语义。
- 约束：同一段落行内代码不超过 3 处；下划线标记每屏不超过 2 处。

### 5. 引用/金句块

**设计方向**：左侧竖条经典款为主，搭配极简渐变底色款。

- **变体 A — 左侧蓝色竖条（默认）**：`border-left:3px solid #2563eb;padding:8px 0 8px 12px;background:transparent`。文字 `15px;font-weight:bold;color:#1a1a2e`。最克制、最常用。
- **变体 B — 浅蓝底卡片（突出强调）**：`background:#f5f7fa;padding:14px 16px;border-radius:6px`。用于文章核心观点或结论性金句。
- **变体 C — 无底色+大引号（极简）**：仅用大号蓝色引号 `"` 作为视觉标记（`font-size:36px;color:#93c5fd`），无底色无边框。适合短金句。
- 选用原则：技术教程用 A；核心结论用 B；少于 20 字的金句用 C。
- 约束：同一文章中引用块不超过 5 个；变体 A 和 B 不同时出现在同一屏。
- **金句提取策略**：AI 应在全文扫描时，识别以下类型的句子作为金句候选：
  - 包含「不是...而是...」「真正...」「本质」等判断句式的句子
  - 以数字开头的结论句（如「3 个关键因素决定了...」）
  - 段落的最后一句（通常是总结性陈述）
  - 长度 15-30 字的紧凑表达
  提取后，短金句（<25 字）用变体 A，中等金句用变体 B，长引用用变体 C。

### 6. 提示/旁注块

**设计方向**：左侧竖条，无顶部标签（极简派）。

- 结构：`border-left:3px solid #3b82f6;background:#f5f7fa;padding:10px 14px`。
- 文字：`font-size:13px;color:#555;line-height:1.75`。
- 可选加前缀 `TIP:` 或 `注意：` 用蓝色 `#2563eb` bold 小字（10px），但非必须——极简蓝倾向于不加标签，让内容本身说话。
- 变体方向：可去除底色仅保留竖条；可改用灰色竖条 `#e2e8f0`（降低层级感）。
- 约束：提示块内不超过 3 行文字；同一屏不超过 1 个提示块。

### 7. 数据/步骤展示

**设计方向**：蓝白数字卡片 + 圆形编号步骤。

- **数据展示**：三列 flex 卡片，浅蓝底 `#f5f7fa`。大数字用 `#2563eb` bold 28px，标签用 `#999` 10px。数字是蓝色锚点——读者扫过时自然停在大数字上。
- **步骤（横向）**：圆形蓝色编号（`width:28px;height:28px;background:#2563eb;color:#fff;border-radius:50%`）+ 步骤名 + 简短描述。
- **步骤（纵向）**：编号左对齐，右侧标题+描述。步骤间用 1px `#e2e8f0` 分割线或留白区分。
- **表格数据**：flex 模拟行，偶数行浅灰底 `#f9fafb`，表头白底+蓝色文字。
- 约束：数据卡片圆角统一 6px；编号始终是蓝色实心圆（不镂空、不用其他色）。

### 8. 图片与媒体

**设计方向**：白底容器 + 极浅灰边框，无阴影（保持干净）。

- 标准图片：白底 `#fff` 容器 + `border:1px solid #f0f0f0`。不使用 box-shadow——极简蓝不需要阴影来创建层次，留白和边框已经足够。
- 圆角策略：图片容器 `border-radius:6px`，与卡片统一。
- 图片说明：`font-size:11px;color:#999;text-align:center;margin-top:6px`。
- GIF 角标：浅蓝底 `#f5f7fa` + 蓝色小字 `#2563eb` 胶囊标签。
- 约束：图片上下间距 12-16px；不使用虚线边框；不使用阴影。

### 9. 代码块

**设计方向**：统一深色终端（暗底），与白底正文形成清晰层次。

- 底色 `#1e293b`，顶栏 `#0f172a` + 三色圆点。
- 代码文字 `#e2e8f0`，等宽字体栈。
- 可选：无顶栏极简版（纯 `#1e293b` 底色 + 内边距，省略圆点装饰）。
- 行内代码：浅灰底 `#f1f5f9` + 蓝色文字 `#2563eb`。
- 约束：深色代码块是极简蓝唯一允许的「暗色区域」，提供视觉节奏切换；不使用浅色代码块（浅色代码块会让页面过于平淡）。

### 10. 分割线

**设计方向**：文字装饰型——灰色省略号 `· · ·`。

- 样式：`text-align:center;font-size:13px;color:#d0d0d0;letter-spacing:6px;margin:24px 0`。
- 文字内容：`· · ·` 或 `...` 或 `— — —`。
- 变体方向：可用两个短横线 `--` + 居中小圆点 `·` + `--` 组合；可用纯 1px 细线 `border-top:1px solid #e2e8f0` 在特殊场景（如章节间）。
- 约束：不使用符号装饰（`◆` 等过于华丽）；分割线颜色不深于 `#e2e8f0`。

### 11. 尾部互动卡

**设计方向**：极简文字链风格（对应设计规范中的风格 D）。

- 卡片底色：浅蓝 `#f5f7fa`，圆角 6px。
- 引导文案：`font-size:13px;color:#666;text-align:center`。示例：「如果这篇文章对你有帮助」「感谢阅读」——语气克制不煽情。
- 互动按钮：使用极简文字链风格——横向排列，间距 24px，`font-size:11px;color:#666;letter-spacing:1px`。不用实心按钮、不用渐变——极简蓝的读者不需要被「按头」互动。
- 可选风格 B 胶囊渐变（仅当文章内容偏轻松时可放宽）：浅蓝渐变 `linear-gradient(135deg,#2563eb,#3b82f6)` 胶囊按钮。
- 结束语：`font-size:10px;color:#999;margin-top:10px`。一句有温度的结尾。
- 约束：尾部卡片高度不超过 200px；不使用阴影；互动按钮不超过 3 个（点赞/在看/转发）。

## 文章类型到设计策略

| 文章类型 | 设计策略 | 节奏特点 | 重点组件 |
|---------|---------|---------|---------|
| 技术教程 | 蓝色锚点定位关键步骤编号；代码块深色终端；步骤流纵向展开；章节分割清晰 | 均匀节奏，每步一屏；步骤间 32px 间隔 | 章节标题、代码块、纵向步骤流、行内代码 |
| 工具测评 | 数据卡片展示工具参数；蓝色大数字抓眼球；对比用两列卡片；标签徽章区分优缺点 | 信息密度中等，数据卡作为视觉锚点穿插正文中 | 数据卡片、标签徽章、图片容器、金句块 B |
| 知识整理 | 章节标题主导结构；大量使用提示块补充背景；列表式步骤纵向排列 | 节奏舒缓，章间大留白；每章一个核心观点 | 章节标题、提示块、纵向步骤流、引用块 A |
| 观点分析 | 金句块 B 提炼核心观点；正文大段留白制造「思考空间」；首段导语做纲领 | 先慢后快，观点句放大，论证段收紧 | 金句块、导语段、引用块 A、分割线 |
| 深度文章 | 大引号（变体 C）标记关键段落；段落节奏有张有弛；章节标题少而精 | 慢节奏，长段落；每 3-4 段一个视觉锚点 | 章节标题、引用块 C、提示块、图片容器 |
| 人文思考 | 蓝色减量（只用灰阶+留白）；正文节奏舒缓；图片占比高（图文并茂） | 极慢节奏，大量留白；蓝色仅 1-2 处出现在全文 | 图片容器、分隔线、引用块 C、首段导语 |
| 产品深潜 | 数据卡+步骤流组合；蓝色锚点定位关键指标；代码块展示技术细节 | 信息密度高，数据区与文字区交替 | 数据卡片、代码块、章节标题、标签徽章 |

## Markdown 到设计决策指南

| Markdown 元素 | 设计决策 | 自由度说明 |
|--------------|---------|-----------|
| `# 一级标题` | 转为文章主标题，`font-size:20-22px;color:#1a1a2e;font-weight:bold;text-align:center`。可选上方加话题标签 | 字号在 20-22px 间调整；是否居中可由 AI 决定 |
| `## 二级标题` | 转为章节标题组件（蓝色竖条+编号+标题），`font-size:18px` | 竖条高度/标题字号在约束范围内微调 |
| `### 三级标题` | 转为无竖条纯编号标题，`font-size:16px;color:#1a1a2e` | 可加浅蓝色短装饰线或不加 |
| 正文段落 | 转为 `<p>` 标签，14px `#333`，行高 1.85，两端对齐 | 段落间距 8-10px 可微调 |
| `**加粗**` | `<strong>` 直接包裹，颜色不变（`#333`） | 不使用彩色加粗 |
| `*斜体*` | `<em>` 包裹，可用于导语或英文书名 | 斜体中文谨慎使用（可读性差） |
| `` `行内代码` `` | 浅灰底 `#f1f5f9` + 蓝色文字 `#2563eb`，等宽字体 13px | 圆角 3-4px 可微调 |
| `> 引用` | 根据内容长度选变体：短金句（<20字）用变体 C 大引号；中等金句用变体 A 竖条；核心结论用变体 B 底色卡 | AI 判断引用内容性质选择变体 |
| ` ``` 代码块 ``` ` | 统一深色终端风格 `#1e293b` 底色，顶栏三色圆点可选 | 顶栏可省略；圆角 6-8px 可微调 |
| `- 列表 / 1. 有序列表` | 纵向步骤流形式（圆形编号+内容），或简洁的 `<p>` + 缩进 | 有编号的用步骤流；无编号的用简洁段落 |
| `--- 分割线` | 灰色 `· · ·` 文字装饰型 | 具体符号（`· · ·` / `...` / `— — —`）可微调 |
| `![图片](url)` | 白底容器 `#fff` + 浅灰边框 `#f0f0f0`，圆角 6px，无阴影 | 圆角 4-8px 范围可调 |
| `[链接](url)` | 转为蓝色下划线文字 `border-bottom:2px solid #93c5fd`（公众号不支持外链，仅作视觉标记） | 下划线颜色固定为 `#93c5fd` |
| 表格 | 用 flex 模拟行，偶数行浅灰底 `#f9fafb`，表头蓝字白底 | 表头可用浅蓝底替代白底 |

## 平台铁律

- 所有中文文字必须用 `<span leaf="">` 包裹
- 禁用 `<div>/<style>/<script>/class/id/position:absolute/float/@media/grid/var()`
- 每个元素必须携带 `box-sizing:border-box;` 和 `max-width:100%!important;`
- `<p>` 标签必须 `margin:0;padding:0;box-sizing:border-box;`
- 所有 CSS 必须内联
- 最大宽度 677px，正文字号 14px（不可改）

## 快速气质校验清单

排版完成后，AI 应自查以下问题：

- [ ] 蓝色是否仅出现在锚点位置（章节编号/竖条/按钮/链接下划线）？没有大面积蓝色底色？
- [ ] 留白是否充分？章节间 32px、组件间 20-24px、段落间 8-10px？
- [ ] 是否有超过 2 个蓝色元素出现在同一屏？
- [ ] 卡片是否统一 6px 圆角？是否无阴影？
- [ ] 分割线是否使用了文字装饰型（非实线）？
- [ ] 代码块是否为深色终端？
- [ ] 尾部互动卡是否使用极简文字链风格？
- [ ] 所有中文文字是否 `<span leaf="">` 包裹？
- [ ] 正文是否 14px / 行高 1.85 / 字间距 0.3px？

## 常见设计陷阱与避免模式

以下是 AI 在使用极简蓝主题进行排版时最常犯的错误，请逐条对照检查：

**陷阱 1：蓝色滥用**——在标签、图标、装饰线、渐变、卡片底纹等多处同时使用蓝色。极简蓝的蓝色是信号灯不是墙漆——全文蓝色出现次数应可数（不超过 8-10 处）。

**陷阱 2：阴影洁癖失控**——极简蓝不使用 `box-shadow`，但 AI 有时会习惯性地给卡片加阴影。如果发现任何 `box-shadow`，请删除并改用 `border:1px solid #e2e8f0`。

**陷阱 3：代码块颜色错误**——使用浅色代码块 (`#f6f8fa`) 而非深色终端 (`#1e293b`)。极简蓝的唯一暗色区域就是代码块，浅色代码块会让全文失去唯一的视觉深度锚。

**陷阱 4：标签全实心**——将所有标签都做成蓝色实心药丸。应区分：话题标签用蓝底白字，分类标签用灰边框+灰色文字。全部实心会让蓝色占比超标。

**陷阱 5：尾部过于热情**——使用风格 B 胶囊渐变按钮。极简蓝的尾部应该克制：文字链风格或浅色边框按钮。想象读者是技术人员——他们不需要被「推销」互动。

**陷阱 6：段落首字下沉**——极简蓝不做首字下沉。段落平等对待，靠留白区分层级。

**陷阱 7：分割线过重**——使用 `border-top:1px solid #333` 之类深色实线。分割线必须是浅灰色文字装饰 `· · ·`，颜色不深于 `#d0d0d0`。

## 排版决策流程速查

AI 拿到一篇 Markdown 文章后，按以下决策链进行排版设计：

```
1. 扫描文章，判断文章类型 → 查「文章类型到设计策略」表确定节奏和重点组件
2. 扫描文章，提取金句 → 按金句长度选引用变体（<25字 A / 25-40字 B / >40字 C）
3. 扫描文章，识别数据点 → 按「数据可视化决策树」选展示形式
4. 生成封面信息卡 → 根据文章气质选方向（技术→档案式 / 知识→极简标题式 / 测评→标签云式）
5. 遍历章节生成章节标题 → 统一使用左侧蓝竖条+编号+标题
6. 遍历段落生成正文 → 14px #333 行高 1.85，每段 5-6 行
7. 遇到代码块 → 深色终端 #1e293b
8. 遇到图片 → 白底+灰边框，无阴影
9. 生成分割线 → · · · 灰色文字装饰
10. 生成尾部互动卡 → 极简文字链风格
11. 全文自检 → 对照「快速气质校验清单」
```
