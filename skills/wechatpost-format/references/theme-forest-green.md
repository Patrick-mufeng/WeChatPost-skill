# 森语绿 -- 设计语言定义

> 本文件是**森语绿**主题的设计语言定义。它为 AI 提供一套完整的视觉决策框架，使 AI 能够理解"森语绿"的气质内核，并据此为每篇文章进行原创排版设计。本文件中的组件描述是**灵感参考**而非复制粘贴模板 -- AI 应在理解设计原则后自主演绎。


## 一、设计变量速查表

```
主色：       #4a7c59（森林绿 -- 只出现在章节编号、极少锚点，全文主色曝光面积极低）
辅色：       #8ba78a（鼠尾草灰绿 -- 辅助装饰、引号、弱化标记）
浅底：       #f5f7f3（卡片/引用块/信息卡底色 -- 极浅，接近白但带一丝草木绿意）
浅边框：     #d8ddd4（柔和的灰绿边框 -- 用于卡片描边、分割元素）
强调点缀：   #6b8f6f（略亮于主色的苔绿 -- 行内高亮底色、关键锚点）
标题色：     #2c3a2e（深墨绿 -- 章节标题、重点标题）
正文色：     #3a4a3e（深灰绿 -- 不纯黑，像铅字印在手工纸上）
辅助文字：   #7a857b（中灰绿 -- 次要信息、元数据、说明文字）
注释色：     #9ca89d（浅灰绿 -- 日期、编号、最弱文字层级）
分割线色：   #dfe3dd（极淡灰绿 -- 分割线几乎消失在底色中，或直接省略）
正文字号：   14px（不可改）
正文行高：   1.85
段落间距：   12px（比常规 8px 多 50%，大留白是核心特征）
章节标题字号：18-20px
章节间距：   40px 上（比常规 32px 多 25%）
字间距：     0.3px
最大宽度：   677px
字体栈：     -apple-system,BlinkMacSystemFont,'PingFang SC','Microsoft YaHei',sans-serif
页面底色：   #fafbf8（暖白带极微绿意，像手工棉纸）
正文关键词下划线 CSS：border-bottom:1.5px solid #b5c8b0;font-weight:500;
```


## 二、设计原则与气质

### 气质关键词
**自然** **侘寂** **留白** **安静** **手工感**

### 配色哲学

这是一套"减法"色彩系统。主色 #4a7c59 只在最必要处出现 -- 章节编号的数字、极少数（全文不超过 3 处）关键锚点。它的存在感像林中偶然瞥见的一片苔藓：你知道它在，但它不喧哗。灰阶承担了 90% 的视觉信息传递任务：从深灰绿正文到浅灰绿注释，六个灰度层级构成阅读秩序。留白不是"空"，而是内容的容器 -- 段落间 50% 的额外间距让每个段落都像独立呼吸的句子。卡片底色 #f5f7f3 与页面底色 #fafbf8 的色差极小（Delta E < 3），层次感来自极微妙的冷暖差而非明暗差。

**色彩分布比例**：页面底色 85% / 卡片浅底 8% / 文字灰阶 6% / 主色 ≤1%。主色在整个页面中的面积占比不超过 1% -- 这是森语绿区别于所有其他主题的核心数字。

### 视觉节奏

- **段落节奏**：段落间距 12px 替代常规 8px。短段落（1-3 句）单独成段，不合并。每 3-5 段插入一次留白呼吸（通过章节标题或引用块制造停顿）。长段落（超过 5 行）应在中间用自然断句拆分。
- **焦点密度**：每屏（约 600px 高度）最多 1 个视觉焦点（锚点/引用/图片）。大多数屏只有一个安静的正文段落。如果连续 3 屏没有焦点，可以放一个浅底卡片或引用块作为温和的视觉停顿。
- **留白策略**：章节之间 40px 间距。卡片上下各 20px 呼吸。页面顶部和底部各留 16px+ 空白。分割线尽量省略，用足量的上下间距替代分割线表达"分隔"语义。列表项之间 12px 间距（而非常规 6-8px）。
- **阅读感**：像翻一本手工纸书 -- 翻页有阻力感，每页信息密度低，目光停留时间长。读者不是在"扫读"，而是在"散步"。

### 强弱规则

| 层级 | 定义 | 频率限制 | 色彩角色 |
|------|------|---------|---------|
| 锚点层 | 章节编号数字、全文金句锚点 | 每章 1 次，全文 ≤3 处 | 主色 #4a7c59 |
| 标记层 | 行内加粗、关键词下划线、引用左竖条 | 每段 ≤1 处加粗 | 辅色 #8ba78a / 正文色加粗 |
| 容器层 | 卡片背景、引用块、信息卡 | 每 3-5 段出现 1 次 | 浅底 #f5f7f3 |

**核心原则**：主色像露水一样稀少。如果读者快速滑动，主色应该只在 2-3 个位置被注意到。其余全是文字本身的质地。

### 设计禁区

- 不要在主色背景上放白色文字 -- 森语绿没有"按钮"，只有"标记"
- 不要在同一个视觉区域内使用 3 种以上颜色
- 不要使用阴影（box-shadow）-- 纸张不需要投影
- 不要使用粗分割线（>1px）或装饰符号分割线
- 不要让任何元素的圆角超过 8px


## 三、组件设计模式

### 1. 封面/头部信息卡

**设计方向**：**手帐便签式** -- 像翻开书时飘出的一张手写便条。

**设计原则**：不需要硬边框。用极浅底色 #f5f7f3 + 微圆角（6px）形成柔和的卡片感。标签用小号灰色文字（非药丸，不用主色背景），跟在"#" 后面像手写标签 -- 如 `#随笔 #自然 #冥想`，每个标签用注释色，中间空格分隔。字数和阅读时间用小号注释色文字（10px），放在一行，用"·"分隔 -- 如 `约 2400 字 · 阅读 6 分钟`。概括句用正文色 + 稍大字号（15px），独段居中，像写在便签上的那句话。

**使用场景判断**：
- 散文/随笔/个人叙事：必用，且概括句偏诗意
- 教程/方法类文章：仍使用，但概括句改为功能描述
- 极短文章（< 800 字）：可省略，直接以首段开场

**约束边界**：不用主色做标签背景。不用阴影。整体像一个淡色的纸片。卡片 padding 不小于 16px，上下的外部间距不小于 12px。

### 2. 章节标题

**设计方向**：**编号+留白** -- 不依赖装饰元素，编号本身就是装饰。

**设计原则**：章节编号使用主色 #4a7c59，字号 18-20px。编号与标题文字之间用两个全角空格分隔（创造视觉停顿）。标题用标题色 #2c3a2e，不加粗（font-weight: normal），靠字号差异和留白建立层级。章节上方留 40px 间距，下方留 8px。不使用左边框装饰条 -- 在森语绿中，装饰条是多余的噪音。

**变体方向**：
- **标准章节**：`01  章节标题`（编号与标题同行）
- **编号独行式**：编号独占一行（字号 22px）+ 标题在下一行（字号 18px），间距 4px。适合标题较短的章节
- **尾声章节**：省略编号，仅用标题文字 + 下方一条极短的细线（30px 宽，分割线色 #dfe3dd），表达"收束"的语义

**约束边界**：不用左边框。不用背景色块。编号不加粗。章节标题不使用主色之外的任何彩色。

### 3. 正文段落

**基准样式**：14px / 1.85 行高 / 0.3px 字间距 / 正文色 #3a4a3e / margin: 0 0 12px 0（注意 12px 底边距是常规的 1.5 倍）。

**首段处理**：首段字号可升至 15px，颜色使用标题色 #2c3a2e，weight 500 -- 像一个温和的开场白，比正文稍重但远不到标题的力度。这不是规则，是一个"建议" -- AI 根据文章调性决定是否采用。如果文章的开头是一段引文或场景描写，首段加大特别合适；如果是直入主题的论述，保持 14px 即可。

**段落对齐**：两端对齐（text-align: justify），适合中文阅读。诗歌、引用、对话类段落可左对齐。独句成段的文字（如过渡句）可用居中。

**段落内部换行**：使用 `<br/>`，换行后的文字保持段落样式。但在森语绿中，尽量用独立段落代替 `<br/>` 换行 -- 多一个段落就多 12px 留白，这符合森语绿的大留白基因。

### 4. 行内标记

- **加粗**：使用 `<strong>` 标签，颜色保持正文色 #3a4a3e，weight 600。不使用主色做加粗颜色 -- 森语绿中加粗是"声音稍大一点"，不是"换一种颜色"。每段最多 1 处加粗。
- **高亮/标记**：使用 `background: rgba(138,167,138,0.15)`（辅色的 15% 透明度），padding: 1px 2px，无圆角。像用淡色荧光笔轻轻划过的痕迹。全文使用不超过 5 处。
- **下划线**：关键词下划线使用 `border-bottom: 1.5px solid #b5c8b0`，font-weight: 500。下划线色是较亮的鼠尾草绿，在正文灰绿中安静地标记重点。不用 text-decoration: underline -- 用 border-bottom 可以精确控制线的粗细和颜色。
- **行内代码**：灰绿底 #eef1ec + 正文色文字，等宽字体，padding: 1px 5px，圆角 3px。不使用深色代码风格 -- 深色代码在森语绿中太突兀。
- **删除线**：使用注释色 #9ca89d，line-through -- 像轻轻划掉一个念头。
- **斜体**：中文极少使用斜体。若出现英文斜体，用 `<em>` + 保持正文色。

**约束**：全文不使用主色 #4a7c59 做行内标记色。主色只属于锚点层。

### 5. 引用/金句块

森语绿的引用块有三种变体方向，按使用场景和"视觉重量"排序。

**变体 A -- 大引号式（视觉重量：轻。首选，文艺静谧）**：
大号引号（字号 36px，辅色 #8ba78a）独占一行，line-height: 0.6 让引号悬在文字上方。金句文字 15px + 正文色，引号与文字间零间距。引号像落在段落上方的一片叶子。适合散文中最有诗意的那句话。全文使用 1-2 次。

**变体 B -- 留白居中（视觉重量：极轻。极简诗意）**：
无引号、无边框、无底色。15px 正文色文字居中，上下各 24px 留白。靠"空"来标记这句话的特别。适合短句（≤15 字）、哲思型的句子。不适合长引用。

**变体 C -- 浅底卡片（视觉重量：中。温和引用）**：
浅底 #f5f7f3、左边框 2px 辅色、padding: 14px 16px。文字 14px 正文色，行高 1.8。适合引用他人言论或资料性引用，需要与正文有明确视觉区分但不过度强调。可在引用下方用注释色 10px 标注出处。

**选用指南**：变体 A 用于文章中最美的一句话（全文 1-2 次）；变体 B 用于核心观点的诗意表达；变体 C 用于引用外部资料（不限制次数）。

**反模式**：不要在变体 A 中使用主色（主色太珍贵）。不要在变体 B 中放超过 15 字的内容（长文字居中会像诗 -- 如果你想要诗的效果，另当别论）。

### 6. 提示/旁注块

**设计方向**：左竖条 -- 辅色 #8ba78a 的 3px 竖条 + 浅底 #f5f7f3。

**设计原则**：不用顶部标签（"提示""注意"等标签在森语绿中过于吵闹）。竖条本身就是标记。文字使用辅助文字色 #7a857b，字号 13px，行高 1.75。padding: 12px 14px。像正文旁边轻声说的一句话。

**使用场景**：补充说明（"关于这一点，多说一句..."）、温和的警告（"注意：..."）、相关内容推荐。不适合作强警告 -- 森语绿没有"强警告"的视觉语言。

**约束边界**：不用虚线框。不用主色竖条（主色在森语绿中太珍贵）。竖条宽度 2-3px。

### 7. 数据/步骤展示

**设计方向**：**轻卡片 + 文字编号**。

**设计原则**：
- **数据展示**：三列卡片用浅底 #f5f7f3，数字用标题色 #2c3a2e（不用主色 -- 主色只用于章节编号），字号 28px weight 600。标签用注释色 10px。卡片 padding: 14px 10px。卡片之间用留白区分（gap: 16px），不用分割线。
- **步骤展示**：纵向布局，编号用 `01` / `02` 文字形式（非圆形），颜色为辅色 #8ba78a，字号 14px。步骤标题使用正文色 weight 600，描述使用辅助文字色 13px。每步间距 16px。
- **列表展示**：无序列表圆点使用辅色 #8ba78a。有序列表序号使用辅色。列表项间距 10px。

**约束边界**：数据数字不用主色，不用药丸编号，不用实心圆形。一切保持克制。在森语绿中，"展示"不是"炫耀"。

### 8. 图片与媒体

**设计原则**：
- **图片容器**：白底 + 无边框（或极淡的浅边框 #d8ddd4），圆角 4px。阴影省略 -- 森语绿不用阴影。图片 `display:block;margin:16px auto`。
- **图片说明**：注释色 #9ca89d，11px，居中。说明文字与图片之间间距 6px。如果有说明文字，把它当作一个独立的小段落，上下各留 4px。
- **GIF 角标**：用浅底 #f5f7f3 + 辅色文字的轻量标签（字号 8px，padding: 2px 6px，圆角 3px），不用主色药丸。位置在图片容器右上角。
- **待补素材**：虚线框用分割线色 #dfe3dd（1px dashed），底色 #fafbf8，与页面底色融为一体。居中放置 emoji 图标 + 注释色说明文字。padding: 24px。

**约束边界**：图片无阴影。圆角不超过 6px。GIF 角标不用主色。

### 9. 代码块

**设计方向**：**浅色代码块** -- 与森语绿的暖白底色协调。

**设计原则**：底色 #f0f2ed（比卡片底稍深一点），左边框 3px 辅色 #8ba78a。代码文字色 #3a4a3e。内边距: 14px 16px。圆角 6px。每行代码使用独立的 `<p>` 标签（margin: 0），行高 1.6。缩进使用全角空格 `　`。

**顶栏选择**：无顶栏更简洁，符合森语绿气质。如果有顶栏（展示语言标签时），使用浅底顶栏 + 注释色语言标签，不带三色圆点（三色圆点太"技术感"）。

**约束边界**：不用深色终端风格。不用主色做边框。代码块在森语绿中应该是"安静的灰色区域"。

### 10. 分割线

**设计方向**：**省略优先 / 极淡细线**。

**设计原则**：在森语绿中，分割线尽量"消失"。首选方案是用 40px+ 的上下间距代替分割线表达分隔语义 -- 留白是森语绿的分割线。如果必须使用（如文章主题切换、大章节之间），用一条极淡的细线：1px solid #dfe3dd，宽度 60px，居中，上下各 20px 间距。

**变体**：极简点线 `· · ·`（注释色 #9ca89d，letter-spacing: 8px）作为备选 -- 适合在需要"呼吸暂停"但不想用实线时使用。

**约束边界**：不使用主色分割线。不使用粗线（>1px）。不使用装饰符号（菱形、星号等）。分割线在森语绿中的存在感应该趋近于零。

### 11. 尾部互动卡

**设计方向**：**手写便签式** -- 像在最后一页夹了一张小纸片。

**设计原则**：浅底 #f5f7f3，padding: 20px 16px，居中。引导文案用正文色 14px weight 500，语气温暖但不热烈 -- 如"读到这里的你，祝你今天平静"、"喜欢这篇文章吗？"。下方间隔 10px 后放置三个轻量按钮：浅底 #e8ebe5 + 正文色文字，圆角 4px，无阴影，padding: 8px 20px。分别放置点赞/在看/转发。底部一句告别语用注释色 11px。

**互动按钮设计**：使用风格 D（极简文字链 -- emoji + 文字：`👍 点赞  👀 在看  ↗ 分享`）或风格 C（大图标文字 -- 大号 emoji + 小字标签）。不用实心色块按钮 -- 太"重"了，不符合森语绿。

**告别语文案方向**：
- 散文/随笔：偏向宁静日常 -- "祝你今天平静"、"下篇文章再见"
- 教程类：偏向温和鼓励 -- "希望对你有帮助"、"慢慢来"

**约束边界**：不用主色做按钮。不用渐变。整体体温是"温"不是"热"。不要写"求点赞"、"跪求转发"之类的文案 -- 森语绿的气质不允许。


## 四、文章类型 -> 设计策略

| 文章类型 | 设计策略 | 节奏特点 | 重点组件 |
|---------|---------|---------|---------|
| 教程/方法 | 用浅底卡片承载每一步，步骤编号用辅色文字编号。信息密度高于常规森语绿，但仍保持段落间 12px 间距。关键命令用行内代码标记。 | 节奏比纯随笔稍快，步骤之间的留白小于章节之间的留白，形成"快慢快慢"的呼吸。每个步骤是一个"停下来"的节点。 | 步骤展示组件、行内代码、浅底卡片 |
| 盘点/清单 | 每项用浅底卡片独立承载，项目间用留白区分而非分割线。编号用辅色文字。每项标题 weight 600，描述正文色。 | 每项一个"呼吸单元"，项与项之间 16px 间距，像一个缓慢翻阅的清单。读者可以在任一项停下来。 | 浅底卡片、数据展示、文字编号 |
| 观点/议论 | 核心观点提取为居中金句（变体 B），论据用正文。观点句周围留白加大到 24px。每个论据段落的开头句可加粗。 | 观点句是大停顿，正文是流动。张弛交替。每 3-4 段正文插入一个观点句作为节奏锚点。 | 居中金句、左竖条引用、行内加粗 |
| 访谈/对话 | 问句用辅色加粗标记，答句用正文。每个问答对之间留白 16px。人物名字可用标题色 weight 600 + 正文色的回答。 | 问句是节奏点，答句是呼吸。一问一答像一个潮汐周期。长的回答可以拆成 2-3 个短段落。 | 大引号引用、左竖条提示、分段对话 |
| 数据/报告 | 数据数字用标题色，使用三列轻卡片展示关键数据。图说文字用注释色。数据之间的正文段落保持森语绿的安静质感。 | 数据卡是视觉锚点，正文是过渡带。数据卡周围留白加大到 20px。数据报告的节奏比纯随笔稍密。 | 数据展示、图片组件、注释说明 |
| 随笔/叙事 | 全文最"森语绿"的使用方式。段落短、留白多、锚点极少（可能全文只有章节编号是主色）。首段加大字号。引用用变体 A（大引号）。 | 慢。非常慢。像溪水。读者被留白引导而非被信息推动。排版应该"消失"，让文字自己说话。 | 大引号引用、省略分割线、首段加大字号、留白居中的过渡句 |
| 案例复盘 | 结构与随笔不同：更清晰的信息层级。用浅底卡片承载每个案例要点，编号用辅色。每个案例有标题（weight 600）+ 正文描述 + 关键数据行。 | 结构感比纯随笔强，但保持留白基因。每个案例间有明确的"转折呼吸"。案例标题周围留白加大。 | 浅底卡片、步骤展示、提示块、数据展示 |

### 文章类型识别与设计决策流程

AI 在收到文章后，应按以下优先级判断设计策略：

1. **先判断文章气质**：是"安静的"（随笔/叙事/冥想）还是"有结构感的"（教程/案例/盘点）
2. **安静型文章**：减少组件使用，增加留白，锚点降到最低（≤2 处主色）
3. **结构型文章**：使用浅底卡片建立层级，但卡片数量和频率仍低于其他主题
4. **混合型文章**（如"哲思+方法论"）：按章节分别处理，安静章节用纯留白策略，结构章节用卡片策略


## 五、Markdown -> 设计决策指南

| Markdown 元素 | 设计决策 | 自由度说明 |
|--------------|---------|-----------|
| `# 标题`（一级） | 文章主标题：标题色 #2c3a2e，字号 22px，weight 600，居中，上下留白 28px/8px。不加装饰。 | AI 可调整字号的 ±2px 范围，可选加副标题行（注释色 12px）。 |
| `## 标题`（二级） | 章节标题：编号用主色 + 标题用标题色，18-20px，上间距 40px。 | 编号格式（`01` / `一、`）可由 AI 选择。是否独占一行由标题长度决定。 |
| `### 标题`（三级） | 子章节：标题色 16px weight 500，上间距 28px。无编号，无装饰。 | 字号可在 15-17px 范围微调。 |
| `**加粗**` | `<strong>` 标签，正文色 weight 600。不使用主色。每段 ≤1 处。 | 无自由度。加粗就是加粗，不换色。 |
| `*斜体*` | `<em>` 标签，保持正文色。中文谨慎使用 -- 仅用于书名、外来语、轻声强调。 | AI 可判断该处是否适合斜体（中文斜体可读性差，建议优先用加粗或下划线替代）。 |
| `` `行内代码` `` | 浅灰绿底 #eef1ec + 正文色等宽字体，padding: 1px 5px，圆角 3px。 | 底色和圆角可微调。 |
| ` ```代码块``` ` | 浅色代码块：底色 #f0f2ed，左辅色边框，无顶栏或简顶栏。 | 有/无顶栏由 AI 决定。语言标签可选。 |
| `> 引用` | 变体 A（大引号）/ B（居中留白）/ C（浅底卡片）三选一。 | AI 根据引用内容的性质和长度选择最合适的变体。 |
| `- 列表` | 圆点使用辅色 #8ba78a。列表项之间间距 10px。 | 缩进深度可微调。嵌套列表符号用 `-` 或空心圆。 |
| `1. 有序列表` | 序号使用辅色 #8ba78a。列表项之间间距 10px。 | 同无序列表。 |
| `![图片](url)` | 白底容器 + 可选极淡边框 + 4px 圆角 + 无阴影。说明文字用注释色 11px。 | 是否加边框由 AI 判断。圆角可在 2-6px 范围。 |
| `---` 分割线 | 首选省略（用加大间距替代）。如需使用，60px 宽细线居中，或点线 `· · ·`。 | AI 可判断该位置是否真的需要分割线。 |
| `[链接](url)` | 公众号不支持外链。转为行内文字标记（如"详见xxx"），使用下划线样式。 | AI 可决定是否保留链接文字并标注来源。 |
| `| 表格 |` | 转为 flex 卡片列表（禁止 `<table>` 标签）。轻底色区分行。 | 可转为纵向卡片列表或保持横向表格感。 |

**平台铁律**：所有含中文的文本节点必须使用 `<span leaf="">文字</span>` 包裹。禁止 `<style>`、`<script>`、`<div>`、`position: absolute/fixed/relative`。所有元素必须带 `box-sizing: border-box;` 和 `max-width: 100% !important;`。
